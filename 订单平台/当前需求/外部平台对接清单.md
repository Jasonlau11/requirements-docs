## 一、已有真实实现（需替换密钥后可用）

### 1. CSDN OAuth 扫码登录

- **需求来源**：《用户注册与身份认证产品需求说明书》§2.2.2 — CSDN 扫码登录
- **文件**：`opcs-infrastructure/.../external/CsdnAuthClient.java`
- **对接方**：CSDN 开放平台（`passport.csdn.net`）
- **实现状态**：✅ 已实现，HTTP 调用完整，未配置时自动降级为 Mock
- **上线待办**：`client-id / client-secret` 当前借用 `csdn_meeting` 项目 key，**上线前须向 CSDN 开放平台申请 opcs 独立应用**


| 配置项           | 环境变量                       | 当前默认值                          |
| ------------- | -------------------------- | ------------------------------ |
| OAuth 服务地址    | `CSDN_OAUTH_SERVER`        | `https://passport.csdn.net`    |
| Client ID     | `CSDN_OAUTH_CLIENT_ID`     | 借用 csdn_meeting                |
| Client Secret | `CSDN_OAUTH_CLIENT_SECRET` | 借用 csdn_meeting                |
| 回调地址          | `CSDN_OAUTH_REDIRECT_URI`  | `localhost:8080/...`（需替换为生产域名） |

### 4. 对象存储（文件上传）

- **需求来源**：
  - 《用户注册与身份认证产品需求说明书》§2.10 — 企业认证文件上传（营业执照、法人身份证、对公账户证明）
  - 《里程碑确认与交付验收产品需求说明书》— 里程碑交付物附件上传
  - 《订单广场与报名竞标产品需求说明书》— 三方协议 PDF 上传
- **文件**：`opcs-infrastructure/.../external/OssClient.java`
- **对接方**：待定（阿里云 OSS / 腾讯云 COS / MinIO 等）
- **实现状态**：✅ 本地磁盘存储（`./uploads/`），文件通过 `GET /files/**` 访问；上线前替换为 OSS 实现

**当前本地存储配置：**

| 配置项 | 环境变量 | 当前默认值 |
| --- | --- | --- |
| 上传目录 | `LOCAL_UPLOAD_DIR` | `./uploads` |
| 访问 URL 前缀 | `LOCAL_ACCESS_URL_PREFIX` | `http://localhost:8888/files` |

**上传接口：** `POST /api/v1/files/upload`（`FileController`，需 JWT 认证）

**对接 OSS 时：** 替换 `OssClient.java` 实现，删除 `local.storage.*` 配置，`WebMvcConfig` 中移除 `/files/**` 静态资源映射。

## 四、不在对接范围内的说明

| 项目       | 原因                                                                                                                                 |
| -------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| 电子签章服务   | 整体方案 v2.4 明确：三方协议流程为「客户上传协议文件 → 接单方确认 → 平台运营审核通过」，**非平台内第三方电子签名服务**。`EsignClient.java` 已标记 `@Deprecated`，若后续引入电子签专项在独立需求中约定。 |
| 支付系统     | 《账单与结算产品需求说明书》明确**一期为线下结算**（客户声明付款 → 接单方确认收款），支付系统对接为**二期**。`PaymentServiceImpl` 存根保持现状，二期替换实现即可。 |
| Redis    | 系统内部技术栈，不属于第三方平台对接。                                                                                                              |
| JWT 密钥   | 系统内部配置，不属于第三方平台对接。                                                                                                               |

## 五、对接优先级


| 优先级       | 平台           | 说明                        |
| --------- | ------------ | ------------------------- |
| P0 — 上线必须 | CSDN OAuth   | 影响注册登录核心流程，仅需替换 key       |
| P0 — 上线必须 | 火山云短信        | 影响验证码发送，仅需替换 key 和模板      |
| P0 — 上线必须 | 阿里云邮件        | 影响邮件通知，仅需替换账号             |
| P0 — 上线必须 | 对象存储 OSS     | 当前本地存储可用，上线前替换为 OSS 实现     |
| P1 — 核心功能 | 大模型服务        | AI 解析创建订单、AI 生成合同/里程碑建议   |
| P1 — 核心功能 | 开发工具资源       | API Key 发放与回收，当前返回假 key   |
| P2 — 重要功能 | CSDN 站内私信跳转  | 指定接单方沟通入口，需确认私信链接格式       |
| 二期         | 支付系统         | 一期线下结算，二期再对接              |
