## 1. 背景

当前供需双方在 CSDN 平台上缺乏统一的「发单—接单—交付—结算」闭环：有任务需求的企业（客户）与具备能力的个人或团队（用户）难以高效匹配，任务类型、交付标准、里程碑与资源使用也缺少标准化管理。同时，CSDN 代理的各类 AI 与开发工具无法与具体订单绑定，导致资源发放与成本归集难以在订单维度落地。

因此，需要建设一套**订单管理系统**：支持客户按任务类型发布订单、用户在订单广场发现与竞标、双方确认里程碑并逐节点交付验收，并由平台运营完成审核与配置；同时将「开发工具 / AI 资源」与订单绑定，实现按订单发放 API Key/激活码，资源用量查看优先沿用火山云平台现有能力，为后续资源计费与运营分析提供基础。

---

## 2. 价值

- **对 CSDN 平台**：形成可规模复用的任务撮合与交付能力，提升平台对企业和个人开发者的服务深度；通过订单与开发资源的绑定，为代理工具的资源管控与变现提供数据与流程支撑。
- **对客户**：在站内按模板发布多类型任务单，经审核后获得订单广场曝光；**认证时提交营业执照与资质证书等材料，经审核后形成统一客户档案，订单等处复用该档案信息，减少重复填写**；拥有**订单看板与订单管理系统**，可查看本人所发出的全部订单信息及状态；拥有**【我的账单】**页面，可查看本人所发出的订单相关的各种账单信息；可从多名用户中择优合作，并通过里程碑确认控制交付节奏与质量。
- **对用户**：统一入口浏览、筛选、报名/竞标订单，可**关注**已发布订单并在**【我的订单】**中按页签（已关注的订单、已接订单、交付中的订单、**已验收订单**（待结算订单、已结算订单）等）查看相关订单；承接订单后原则上须完成**订单三方协议**在平台上的**确认与生效流程**（客户上传协议文件、接单方确认、平台审核通过），原则上与 **CSDN 结算**；拥有**【我的账单】**页面，可查看本人参与每一个订单所产生的各种账单信息；展示个人或团队能力与资质；按里程碑交付并获取与订单绑定的开发资源，流程清晰、资源可追溯。
- **对运营方**：集中审核客户提交的订单内容合规性，配置任务类型与发布模板、订单可选/必选开发工具；开发资源用量由运营在火山云平台侧查看。运营端拥有**业务看板**，可一站式查看当前平台的**客户**、**用户**、**订单**、**账单与结算**、**合同**、**后台设置**等核心内容，支撑运营与资源策略。

## 4. 范围与边界

### 4.1 本期包含

- **用户注册与身份**
  - **统一入口与登录**：使用订单广场前需先在**独立部署网站**完成注册/登录；支持通过 **CSDN App 扫码**快速创建网站账号并完成账户绑定；所有账户从**统一入口**进入【订单广场】页面。
  - **首次进入订单广场**：账户**第一次**进入订单广场时，若该账号**尚未选择身份**，则需**选择身份**（**客户**或**用户**，二选一）；选择后进行**对应资质认证**。**客户**需完成**企业资质认证**（提交营业执照与资质证书等材料并通过审核）；**用户**需完成**个人/团队能力与资质**的填写与维护。**认证完成后**，再次进入订单广场**不再弹出**身份选择或认证页面。
  - **单一身份**：同一账号**仅能具备客户或用户身份之一**，不可同时具备两种身份。**客户**：仅具备发单能力，在订单广场可见【发布订单】按钮并可发单，**不可接单**；**用户**：仅具备接单能力，在订单广场**不可见【发布订单】按钮**，可接单。
  - **客户（发单）身份**：选择客户身份并完成企业资质认证后，该账号即固定为客户身份。企业资质认证通过**上传营业执照与资质证书**完成；其中支持通过**上传营业执照解析自动填充认证信息**（具体字段与规则在需求文档中细化）。审核通过后形成该客户在平台上的统一信息档案；**订单及各处展示的客户相关信息复用该档案，减少客户重复填写**。认证材料清单、审核规则与人工复核流程在需求文档中细化。
  - **用户（接单）身份**：选择用户身份并完成用户资质（个人/团队能力与资质）补充后，该账号即固定为用户身份。需填写或维护个人/团队能力背景、资质证书等信息；可在身份选择后立即进入资质补充流程完成认证。
  - 运营方为平台运营人员，使用既有账号与权限体系，不通过上述身份选择与资质认证流程开通。

- **订单发布与审核**
  - **任务类型**：客户发布的订单类型固定为以下 **8 种**：文本创作、视频创作、软件开发、产品评测、营销推广、平面设计、**硬件订单**、模版创建。其中**模版创建**仅运营端可创建，客户端不可选择该类型；**模版创建**为**内部自用任务**，不需要用户承接流程，**仅支持运营端承接**，**不走里程碑与结算**，该两步可直接跳过，采用**简化流程**（如审核通过后运营方承接即视为完成，具体状态流转在需求文档中定义）。
  - **订单创建模式**：每一种任务类型均支持 **三种创建模式**，且可与**订单模板**结合使用——客户（或运营方）可先选择任务类型、可选选择某订单模板，再选择一种创建模式并完成创建：
    - **手工创建**：按模板或默认表单项逐项填写订单内容。
    - **上传文档由 AI 解析创建**：上传文档（如 Word、PDF），由后台 AI 解析文档内容并生成订单草稿，客户确认或修改后提交。
    - **自然语言 AI 解析创建**：客户用自然语言描述需求，由后台 AI 解析并生成订单草稿，客户确认或修改后提交。
  - 任务类型及对应发布模板由运营在后台配置；客户选择任务类型（前 7 种之一）、可选选择该类型下的订单模板，再结合上述三种创建模式之一填写或生成订单内容。
  - **订单必填信息**：客户发布订单时需填写**结算模式**、**订单交付模式**、**订单报价**、**接单要求**等信息（具体字段与校验规则在需求文档中定义）；可与模板字段、创建模式结合，由手工填写或 AI 解析生成后确认。**结算模式**在客户定义订单时确定，分为两种：**① 平台抽佣**：客户与用户直接交易结算，由**客户**定义该订单的里程碑、结算等内容；**② 平台承包转包**：客户将订单承包给平台，平台再转包给用户，由**平台运营方**定义该订单的里程碑、结算等操作。客户新建订单时需配置**响应时限/招标时限**：**指定接单方**的订单配置**响应时限**（指定方需在时限内响应或拒绝），**公开抢单**的订单配置**招标时限**（订单在订单广场接受报名的时长）；支持常用时限预设（如 3 天、7 天、15 天等）或**无限制**，具体选项在需求文档中定义。
  - **订单属性与接单方式**：**订单属性**为**统称**，包含**订单性质**（私密订单/公开订单）、**预算区间**、**工作量**等其他属性（具体枚举与定义在需求文档中明确）。客户创建订单时可设置**订单性质**为**私密订单**或**公开订单**，并选择接单方式：**私密订单**可**指定接单方**（指定某用户）或**指定由平台运营方接单**；**公开订单**可**指定接单方**或**公开抢单**（在订单广场公开展示，供用户报名/竞标）。仅「公开 + 公开抢单」的订单在订单广场展示；指定接单方的订单仅对被指定用户可见/可接，私密且运营方接单的订单由运营方在后台承接。
  - **上架状态**：订单在**推广中**阶段需区分**已上架**与**未上架**（如指定接单方订单在指定方尚未确认接单前为未上架，公开抢单订单审核通过并展示于订单广场后为已上架），具体判定规则与展示逻辑在需求文档中定义。
  - **响应时限/招标时限与超时行为**：**指定接单方**的订单可配置「时限内未响应或拒绝」时的处理方式：**自动下架**或**自动转为公开抢单**（转为公开后按公开抢单规则与招标时限执行）。**公开抢单**的订单可配置招标时限超时后的处理方式：**自动下架**，或**超时订单自动隐藏**（不再在订单广场展示，效果与下架类似，具体采用何种方式在需求文档中确定）。**响应时限/招标时限**可在订单处于**待审核**、**未上架**（推广中且未上架）、**推广中（已上架）**状态下由客户灵活调整，具体可调整次数与规则在需求文档中定义。
  - **审核与状态**：客户提交订单后由**运营方**进行审核；审核通过则按订单属性与接单方式发布（上架订单广场或对指定方可见），不通过则打回（需反馈原因，客户可修改后重新提交）。订单具备完整**状态流转**，定义如下。
  - **订单状态流转**：
    - **新建**：客户创建订单草稿，未提交审核；可编辑、可删除、可提交。
    - **待审核**：客户已提交，等待运营方审核；运营方可「通过」或「打回」。客户与运营方均可在此阶段取消订单。
    - **已打回**：运营方审核不通过；客户可修改后重新提交，提交后 → 待审核。
    - **推广中**：运营方审核通过，订单已发布，等待确定接单方（区分**未上架**/ **已上架**：如指定接单方尚未确认接单为未上架，公开抢单已在订单广场展示为已上架）。客户与运营方均可在此阶段取消订单。指定接单方在响应时限内未响应或拒绝时，按配置执行自动下架或自动转为公开抢单；公开抢单订单招标时限超时后，按配置执行自动下架或自动隐藏。
    - **协议签署中**：客户选标或指定接单方/运营方确认接单后进入此状态；须就**订单三方协议**（客户、用户、平台为协议关系主体）在平台完成**客户上传协议文件 → 接单方确认 → 平台运营审核通过**，使协议版本**生效**并**自动关联当前订单**。**协议版本生效**是进入**交付中**的必要前置条件（本流程为**上传—确认—审核**的业务闭环，**不采用**平台内第三方电子签名服务，亦**不等同于**「双方仅在页面点击即完成线下同等效力签署」；具体交互见《订单广场与报名竞标》等需求文档）。协议确认超时或一方拒绝时的处理（如取消、退回推广中）在需求文档中定义。
    - **交付中**：协议版本已按上述流程**生效**并关联订单，里程碑进行中（尚未全部通过）；客户与接单方按里程碑提交与确认。**里程碑超期与终止**：客户与用户商定的某个里程碑超过约定期限（如约定完成日 + 宽限期，具体在需求文档中定义）未达成时，**开放客户的终止订单权利**（此前不开放）；客户行使终止权后订单进入**已终止**。是否支持运营方在交付中取消/终止订单（如纠纷、严重违约）在需求文档中确认；若支持，建议限定为运营方操作并需记录原因。
    - **已验收**：全部里程碑已通过，交付完成。
    - **待结算**：已验收，等待结算流程（支付、对账等）；具体规则在需求或后续迭代中定义。
    - **已结算**：结算完成，订单闭环结束。
    - **已取消**：客户或运营方在**待审核**、**推广中**阶段取消订单。取消订单采用**逻辑删除**，报名记录及相关数据保留在数据库中，不影响历史数据查询与审计。
    - **已终止**：**交付中**阶段因里程碑超期等原因，客户行使终止权（或运营方依规则操作）后订单结束。**终止后结算**以客户与用户之间的相关协议为准，由**运营端人工审核**双方提供的合同/协议内容并按条款执行结算；**平台不承诺系统自动结算**，以规避平台法律责任。具体审核流程与结算执行在需求或后续迭代中细化。

- **订单广场与报名/竞标**
  - **订单广场**仅展示**公开且选择公开抢单**的订单；用户在此可按**订单类别**（任务类型）、**订单属性**（含订单性质、预算区间、工作量等）、**交付模式**、**是否无人接单**、**新订单**等进行**分类与搜索**，对感兴趣的订单可**关注**并报名/竞标；客户在报名列表中通过沟通或面试确定一名用户承接。
  - **用户关注订单**：用户可对订单广场上已发布的订单进行**关注**，关注后的订单可在用户侧**【我的订单】**页面的「已关注的订单」页签中查看。
  - **指定接单方**的订单（无论私密或公开）：仅被指定的用户可见并可接单（通过「指定给我的订单」等入口），不进入订单广场公开展示。
  - **私密且指定由平台运营方接单**的订单：由运营方在后台承接，不向普通用户展示。

- **里程碑与交付验收**
  - 根据订单的**结算模式**，里程碑与总工期由**客户**（平台抽佣模式）或**平台运营方**（平台承包转包模式）与用户共同确认；支持自定义，如总工期 3 个月、每月一个里程碑及对应交付内容。
  - 各里程碑节点：用户提交交付物 → 客户确认；客户确认通过则该里程碑通过。
  - 全部里程碑通过后，该订单进入正式结算（结算规则与支付流程在需求文档或后续迭代中细化）。

- **协议与结算原则**
  - **订单三方协议与协议签署中**：选标或指定接单方/运营方确认接单后，订单进入**协议签署中**（状态名保留；业务含义为协议**确认与生效流程进行中**）。**订单三方协议**指客户、用户、平台为权利义务主体的协议文本；在系统中通过**客户上传协议文件 → 接单方（用户或运营方身份）确认 → 平台运营审核通过**使协议版本**生效**并**自动关联当前订单**（支持客户使用 AI/模板生成后线下定稿再上传，或直接上传已在站外形成/签署的协议文件，详见《订单广场与报名竞标》等需求文档）。**协议版本生效**是订单进入**交付中**的必要前置条件；**非**平台内第三方电子签或「在线缔约」产品能力。协议确认超时或一方拒绝时的处理（如取消、退回推广中）在需求文档中定义。
  - **结算模式与定义方**：客户在定义订单时需确定**结算模式**。**① 平台抽佣**：客户与用户直接交易结算，该订单的里程碑、结算等内容由**客户**定义；**② 平台承包转包**：客户将订单承包给平台、平台再转包给用户，该订单的里程碑、结算等操作由**平台运营方**定义。
  - **结算关系**：原则上**用户与 CSDN 结算**，即结算主体为用户—CSDN；客户与平台、平台与用户的资金与结算规则在需求或后续迭代中补充。
  - **订单终止与终止后结算**：客户与用户商定的某个**里程碑超过约定期限未达成**时（如约定完成日 + 宽限期，具体在需求文档中定义），**开放客户的终止订单权利**，此前不开放。客户行使终止权后订单进入**已终止**。**终止后的结算以客户与用户的相关协议为准**，由**运营端人工审核**双方提供的合同/协议内容并按条款执行结算；**平台不承诺系统自动结算**，以规避平台法律责任。具体审核流程与结算执行在需求或后续迭代中细化。

- **账单与结算（订单关联）**
  - **设计说明**：采用「**按阶段结算可配置 + 结算动作灵活发起**」：账单生成可依里程碑达成或订单待结算触发，**结算动作**（确认、支付、入账）**不绑定在里程碑完成时刻**，由客户、用户或运营在账单就绪后按权限灵活发起。这样既支持按阶段出账、部分结算与剩余取消，又避免在系统内 1:1 复刻合同中的结算时点，兼顾灵活性与实现成本。
  - **订单与账单的关系**：订单与账单是**独立对象**，二者通过关联关系与状态协同。当订单转为**待结算**时，系统**自动创建账单**并置于账单**初始状态**（见下「账单状态」）；同时支持**一订单多账单**（按阶段结算时，每个阶段/里程碑对应独立账单）。
  - **订单状态与账单状态的关系**：**订单状态**表示**业务进度**（新建→待审核→…→待结算→已结算等）；**账单状态**单独定义（见下），表示结算单据自身的生命周期。二者**有关联**：订单转为**待结算**时自动产生账单并置为初始状态；订单进入**已结算**的判定可与该订单下全部应结账单的结算完成情况挂钩（见「用户侧收入账单与订单已结算」）。若**账单被冲销/重开**，订单状态是否回退、如何回退在需求文档中定义。
  - **账单状态**（单独定义，与订单状态协同使用）：**待确认**（已生成、待双方或运营确认金额与条款）、**待支付**（已确认、等待支付）、**支付中**（支付流程进行中）、**已支付**（支付完成）、**已入账**（资金已入账、结算完成）、**已冲销**（作废或冲销）、**已关闭**（取消结算、不再结算）。具体状态枚举与流转在需求文档中定义。
  - **待结算的含义**：订单侧**待结算**表示「交付已完成或已终止，进入结算阶段」；可对应「已生成结算单/账单并等待支付」或「等待线下财务动作」，具体以需求文档为准。订单进入待结算时，可自动生成尾款账单（整单结算）或仅标记订单进入待结算（阶段结算时由前述阶段账单已逐笔生成）。
  - **按阶段结算（可配置）**：支持将订单配置为**按阶段结算**。配置后：**交付中**即可产生账单——每达成一个里程碑（客户确认通过），可自动生成该阶段账单并置于初始状态（如待确认）；订单**全部里程碑通过**转入**待结算**时，再生成**尾款账单**。未配置按阶段结算时，仅在订单进入待结算时生成**整单账单**（一次）。是否按阶段结算可在订单创建或模板中配置，具体在需求文档中定义。
  - **结算动作的发起**：**结算动作**（将账单从待确认/待支付推进至已支付、已入账）**不强制绑定在里程碑完成时刻**；支持**灵活发起**——当账单已生成且处于可结算状态时，由客户、用户或运营方（依权限与规则）发起结算动作，经确认/支付/入账后更新账单状态。这样既支持「按阶段结算」，又支持按双方约定时间发起结算，无需在系统内 1:1 复刻合同中的结算时点。
  - **客户侧「我的账单」**：每条账单与订单关联；**按阶段结算**时为一订单多条账单（如按里程碑或按阶段），**整单结算**时为一订单一条账单。客户侧展示本人所发订单下的应付、支付状态、结算状态等，具体展示项在需求文档中定义。
  - **用户侧收入账单与订单已结算**：用户侧收入账单与订单关联；当该订单下**全部应结账单**状态为**已支付/已入账**（或按业务规则视为结算完成）时，订单可进入**已结算**。若存在多笔账单，需全部结算完成后订单才可标记为已结算，具体规则在需求文档中定义。
  - **订单取消/终止与账单**：订单在**待审核、推广中**取消（**已取消**）时，一般不生成应付/应收账单，或仅生成**冲销/零金额**记录以备审计。订单在**交付中**被运营方取消或**已终止**时：已产生的**阶段账单**按以下规则处理——**未结算的账单**可作**已冲销/已关闭**，不再结算；**已结算的账单**若需退款则走退款流程，具体在需求文档中定义。**交付中取消/终止**时，支持**款项原路退回**；并可配置**对用户的补偿机制**（如因不可抗力取消时，按实际工作量支付用户一定报酬，上限由平台或订单约定），具体规则在需求文档中定义。
  - **部分结算 + 剩余取消**：支持**部分结算 + 剩余取消**的混合场景。例如订单有 A、B、C 三个里程碑并配置按阶段结算；A、B 里程碑达成并对应账单已结算成功，C 里程碑未达成且订单被终止——则仅结算 A、B 阶段账单，C 阶段不生成账单或生成后作**已冲销/已关闭**，不再结算。订单状态为**已终止**，结算以双方协议及运营审核为准。
  - **待结算后取消结算**：若订单已进入**待结算**后需取消结算（如发现问题、纠纷），状态可回退为**已验收**或**已终止**（视业务规则）；对应账单做**已冲销**或**已关闭**，并可按需生成退款/冲销账单，具体回退条件与账单更正在需求文档中定义。

- **开发工具与资源绑定**
  - 运营方根据当前代理业务需求，在后台配置某订单**可选工具列表**或必选某一/多个指定工具；**接单方（用户）**可在**【我的订单】-【交付中的订单】**中点击订单、查看订单详情后，**配置该订单采用的交付工具**，从该订单的可选工具列表中**选择一个或多个工具**用于交付（如软件开发订单可选 Cursor + Trae 协同交付）。支持选择的**交付工具**分为多类：文档工具、文档创作工具、原型设计工具、音视频生成工具、开发工具等（具体分类与枚举在需求文档中定义）。**多工具**：支持同一订单**多选多个工具**，各工具**分别统计用量**；运营看板可按订单汇总或按工具维度拆开查看。
  - **资源发放与回收**：**发放时机**为**订单三方协议版本在平台生效**之后（非确认接单后）；用户在订单进入**交付中**后可在该订单的「开发工具/资源」入口获取针对当前订单的 API Key、激活码等。**失效回收**：订单进入**已结算**或**已取消**（含**已终止**）后，绑定资源**自动失效/收回**。**交付中切换交付工具**：同一订单支持在交付中**切换或增删**所选交付工具；切换后**之前工具的资源优先失效**，若下游无法做到随时控制资源失效则**保持可用**；该订单的资源用量统计**订单交付周期内的完整用量**（含切换前后各工具用量），按工具分别统计。
  - 后台按订单绑定的开发资源、**按工具分别**统计该订单的实际消耗用量（如 API 调用量、token 等），运营方可在后台按订单或按工具查看消耗统计。
  - **工具与资源的采购与来源**（业务规则层面，具体执行与渠道在需求或商务流程中细化）：① **部署的云**须为 CSDN 指定云，由 CSDN 购买或委托 OPC 采购（CSDN 提供渠道码）；② **开发工具**（如 TRAE 企业版等）原则上由 CSDN 指定购买（CSDN 提供渠道码、授信）；③ **设计工具、文档工具、音视频制作工具**可根据创作者使用习惯购买，可参与数字游民基地由 CSDN 组织 OPC 的「团购」等方式获取。

- **运营后台**
  - **业务看板**：运营端支持**业务看板**功能，可集中查看当前平台的**客户**、**用户**、**订单**、**账单与结算**、**合同**、**后台设置**等核心内容；各模块支持从看板入口进入对应列表或配置页，具体看板布局、统计维度与跳转逻辑在需求文档中细化。
  - 订单审核（通过/打回）、订单发布模板管理、任务类型管理、开发工具与订单的配置、订单维度开发资源用量统计与查看。
- **客户侧订单看板与订单管理**
  - 客户需拥有**订单看板**及**订单管理系统**（客户端），能够查看本人所发出的**全部订单**信息（含订单状态、类型、报价、接单方式、里程碑进度等）；支持按状态、类型、时间等筛选，进入订单详情后进行报名列表查看、选标、里程碑确认等操作；具体看板视图与列表/筛选能力在需求文档中细化。

### 4.2 本期不包含

- 客户与用户**双身份共存**：本期采用**单一身份**，同一网站账号仅能选择**客户**或**用户**身份之一，**不支持**同一账号同时具备发单与接单能力。**仅运营人员**可同时以“发单”与“承接”角色存在（如创建**模版创建**类型订单并由运营方自己承接），与普通账号的单一身份区分。若业务后续需支持双身份，再在需求中明确。
- 订单支付、担保、发票、合同电子签等资金与法务流程的详细设计，本期仅预留「正式结算」节点，具体规则在后续需求或迭代中补充。
- 开发资源的具体计费规则、与订单金额的抵扣或结算关系，本期不在订单系统建设用量统计聚合与展示页面，不做自动计费与结算。

### 4.3 与现有产品/系统关系

- **账号与身份**：采用**独立部署网站账号体系**与登录态，不复用 CSDN 站内直登体系。网站支持常规注册/登录，并提供 **CSDN App 扫码快速创建网站账号**能力；扫码成功后自动建立 CSDN 账户与网站账户绑定关系。**订单广场**为**统一入口**：网站登录后可进入订单广场。**首次进入**订单广场时，若该账号尚未选择身份，则需**选择身份**（客户或用户，二选一），选择后完成**对应资质认证**（客户为企业资质认证，用户为个人/团队能力与资质），认证完成后再次进入订单广场不再弹出身份选择或认证页面。**单一身份**：同一账号仅能具备客户或用户身份之一；**客户**在订单广场可见【发布订单】并可发单、不可接单，**用户**不可见【发布订单】、可接单。运营方使用平台既有运营/管理员权限。
- **入口与信息架构**：订单管理为新增业务模块，需在「我的」或主导航中增加**订单广场**等订单相关入口。**订单广场**为客户与用户共同入口，所有账户从该入口进入；页面内容大致相同（完整订单列表、点击订单可查看订单详情），**客户**可见【发布订单】按钮并可发单、不可接单，**用户**不可见【发布订单】、可接单。**客户**（已选择客户身份并完成认证的账号）需有**订单看板/订单管理**入口，可查看**本人所发出的全部订单**（含已取消、已终止），并支持**「仅看有效订单」**过滤；需有**【我的账单】**入口。**用户**（已选择用户身份并完成认证的账号）需有**【我的订单】**入口，进入后为「我的订单」页面，页面内包含**指定给我的订单**、**已关注的订单**、**已接订单**、**交付中的订单**、**已验收订单**（下含待结算订单、已结算订单）等页签，指定给本人且尚未确认接单的订单在「指定给我的订单」页签展示，确认接单后归入已接订单等；用户需有**【我的账单】**入口。订单广场、我的订单、我的账单等入口与现有个人中心、设置等并列。**运营端**需有**业务看板**入口。
- **开发工具/代理业务**：依赖 CSDN 现有或规划中的代理工具与资源发放能力；订单系统内优先实现「按订单绑定资源」与发放/失效，不建设用量统计聚合能力，用量查看依赖火山云平台。

## 5. 整体方案

CSDN 订单管理系统定位为**面向客户与用户的任务发布、竞标、里程碑交付与资源绑定平台**：客户按**任务类型**（文本创作、视频创作、软件开发、产品评测、营销推广、平面设计、**硬件订单**）可选选择**订单模板**，结合**三种创建模式**（手工创建、上传文档AI解析创建、自然语言AI解析创建）之一创建订单，并可设置**订单属性**（私密/公开）与**接单方式**（指定接单方、运营方接单、公开抢单），配置**响应时限/招标时限**（指定接单方为响应时限，公开抢单为招标时限，支持预设或无限制）及超时行为（自动下架或转为公开/自动隐藏）；推广中订单区分**已上架/未上架**，时限可在待审核、未上架、已上架状态下由客户调整。经审核后，公开抢单的订单展示于订单广场供用户筛选与报名/竞标，指定接单方的订单仅对指定用户或运营方开放；选标/确认接单后进入**协议签署中**，经**客户上传 → 接单方确认 → 平台审核通过**使协议版本**生效**并关联订单后进入**交付中**。**待审核、推广中**可由客户或运营方取消（**已取消**，逻辑删除、报名记录保留）；**交付中**某里程碑超期未达成时开放客户**终止订单**权利，订单进入**已终止**，终止后结算以客户与用户协议为准、由运营人工审核并按条款执行，平台不自动结算。根据订单**结算模式**，里程碑由客户（平台抽佣）或平台运营方（平台承包转包）与接单方确认后按节点交付验收，全部通过后进入结算；用户在承接订单后可获取该订单绑定的开发资源（API Key/激活码等），资源用量查询由运营在火山云平台查看；运营方在网页端完成订单审核、任务类型与模板配置、开发工具配置及终止订单结算审核，可创建**模版创建**类型订单并可承接私密且指定由运营方接单的订单；运营端支持**业务看板**，可集中查看当前平台的**客户**、**用户**、**订单**、**账单与结算**、**合同**、**后台设置**等内容。

## 6. 功能点拆分与详情

### 6.1 用户注册与身份认证

#### 6.1.1 简述
用户在**独立部署网站**完成注册/登录后使用订单广场，**首次进入**订单广场需**选择身份**（客户或用户二选一），选择后进行**对应资质认证**；认证完成后再次进入订单广场不再弹出身份选择或认证页面。网站支持通过 **CSDN App 扫码**快速创建用户并绑定 CSDN 账户与网站账户。**客户**完成企业资质认证（提交营业执照与资质证书等材料并通过审核）；**用户**填写个人/团队能力背景与资质证书信息。同一账号**仅能具备一种身份**，客户仅能发单、用户仅能接单。

#### 6.1.2 详情 / 关联文档
- **身份与入口**：
  - **统一入口**：订单广场为**客户与用户共同入口**，需先完成网站账号注册/登录后进入；支持通过 **CSDN App 扫码**快速创建网站账号并自动绑定 CSDN 账户；**所有账户从统一入口进入【订单广场】页面**。
  - **首次进入订单广场**：账户**第一次**进入订单广场时，若该账号**尚未选择身份**，则需**选择身份**（**客户**或**用户**，二选一）；选择后进入**对应资质认证**流程。**客户**需完成企业资质认证（提交营业执照与资质证书等材料并通过审核）；**用户**需填写或维护个人/团队能力与资质。**认证完成后**，再次进入订单广场**不再弹出**身份选择或认证页面。
  - **单一身份**：同一账号**仅能具备客户或用户身份之一**。**客户**：仅具备发单能力，在订单广场可见【发布订单】按钮并可发单，**不可接单**；**用户**：仅具备接单能力，在订单广场**不可见【发布订单】按钮**，可接单。完成客户资质后，该账号即固定为客户身份，拥有订单看板、订单管理及【我的账单】等客户侧入口；完成用户资质后，该账号即固定为用户身份，拥有【我的订单】、【我的账单】等用户侧入口。
- **客户认证**：
  - **资质材料提交与审核**：客户提交**营业执照**与**资质证书**等企业认证材料，平台按规则进行审核；支持上传营业执照后解析并自动填充认证信息（具体字段、校验与可编辑范围在需求文档中定义），审核通过后形成结构化客户档案（如企业名称、行业、简介、资质信息等）。
  - **信息复用**：订单发布、订单详情、订单广场等所有涉及「客户相关信息」的展示与数据，均复用认证通过后的客户档案信息，**无需客户在每笔订单或多处重复填写**；若信息需补充或修正，支持客户在认证环节或档案管理中申请更新（具体规则在需求文档中定义）。
  - 认证材料清单、字段定义、审核方式及人工复核规则在需求文档中细化。
- **用户**：个人或团队能力背景介绍、所取得资质证书等信息的结构化填写与展示；在首次进入订单广场并选择用户身份后进入资质补充流程完成认证。与客户身份互斥，同一账号仅能具备一种身份。
- **详见后续《用户注册与身份认证》需求文档。**

### 6.2 订单发布模板与任务类型管理（运营端）

#### 6.2.1 简述
运营在后台配置**任务类型**（固定 8 种）及每种类型对应的**订单发布模板**；每种任务类型均支持与**三种创建模式**结合使用，供客户（或运营方）选择模板后按手工/上传文档AI解析/自然语言AI解析之一创建订单。**模版创建**类型仅运营端可创建。

#### 6.2.2 详情 / 关联文档
- **任务类型（固定 8 种）**：
  - 文本创作、视频创作、软件开发、产品评测、营销推广、平面设计、**硬件订单**、模版创建。
  - **模版创建**仅运营端可创建，客户端创建订单时不可选择该类型。**模版创建**为**内部自用任务**：不需要用户承接流程，**仅支持运营端承接**；**不走里程碑与结算**，该两步直接跳过，采用**简化流程**（如审核通过后运营方承接即视为完成，具体状态流转在需求文档中定义）。
- **发布模板**：按任务类型配置模板字段，须包含**订单交付模式**、**订单报价**、**接单要求**及标题、描述、工作量、交付要求等，支持必填/选填与校验规则；客户（或运营方）可选择某模板后，结合**订单创建模式**之一填写或生成内容。
- **订单创建模式**（与模板结合）：手工创建、上传文档由 AI 解析创建、AI 根据用户自然语言描述解析创建；每种任务类型均支持上述三种模式，需求文档中定义各模式下的表单项与 AI 解析输出字段。
- **详见后续《订单发布模板与任务类型管理》需求文档。**

### 6.3 客户发布订单与审核

#### 6.3.1 简述
客户选择**任务类型**（文本创作、视频创作、软件开发、产品评测、营销推广、平面设计、**硬件订单**之一）、可选选择该类型下的**订单模板**，再选择**一种创建模式**（手工创建、上传文档AI解析创建、自然语言AI解析创建）填写或生成订单内容，并设置**订单属性**（私密/公开）与**接单方式**后提交；运营方审核通过则按属性与接单方式发布，不通过则打回并反馈原因。

#### 6.3.2 详情 / 关联文档
- **发单流程**：选择任务类型（前 7 种之一）→ 可选选择订单模板 → **选择创建模式**（手工 / 上传文档AI解析 / 自然语言AI解析）→ 按所选模式填写或输入（手工逐项填、上传文档由 AI 生成草稿、输入自然语言由 AI 生成草稿）→ 必填**结算模式**、**订单交付模式**、**订单报价**、**接单要求**及**响应时限/招标时限**（指定接单方配置响应时限，公开抢单配置招标时限，支持常用预设或无限制）→ 确认或修改草稿 → **设置订单属性与接单方式**（含指定接单方未响应/拒绝时的处理：自动下架或转为公开抢单；公开抢单超时处理：自动下架或自动隐藏）→ 提交审核（订单状态由「新建」→「待审核」）。
- **订单必填信息**：客户发布订单时须填写**结算模式**、**订单交付模式**、**订单报价**、**接单要求**及**响应时限/招标时限**（指定接单方订单为响应时限，公开抢单订单为招标时限）；**结算模式**二选一：**平台抽佣**（客户与用户直接交易结算，由客户定义该订单的里程碑、结算等）或**平台承包转包**（客户将订单承包给平台、平台转包给用户，由平台运营方定义该订单的里程碑、结算等）。具体字段含义、格式与校验规则在需求文档中定义，可与发布模板字段统一配置。
- **响应时限与招标时限**：客户新建订单时需配置——**指定接单方**订单配置**响应时限**（被指定方需在时限内响应或拒绝），**公开抢单**订单配置**招标时限**（订单在订单广场接受报名的时长）；支持常用时限预设（如 3 天、7 天、15 天等）或**无限制**。该时限可在订单处于**待审核**、**未上架**（推广中且未上架）、**推广中（已上架）**时由客户灵活调整，具体可调整次数与校验在需求文档中定义。
- **上架状态与超时行为**：推广中阶段区分**未上架**/ **已上架**（如指定接单方尚未确认为未上架，公开抢单已展示于订单广场为已上架）。指定接单方订单可配置：时限内未响应或拒绝 → **自动下架**或**自动转为公开抢单**。公开抢单订单可配置：招标时限超时 → **自动下架**或**超时订单自动隐藏**（不再在广场展示），具体采用下架或隐藏在需求文档中确定。
- **订单创建模式**：
  - **手工创建**：按模板或默认表单项逐项填写订单内容。
  - **上传文档由 AI 解析创建**：上传文档（如 Word、PDF），后台 AI 解析并生成订单草稿，客户确认或修改后提交。
  - **自然语言 AI 解析创建**：客户用自然语言描述需求，后台 AI 解析并生成订单草稿，客户确认或修改后提交。
  - 三种模式均可与**订单模板**结合：选择模板后，手工模式按模板字段填；两种 AI 模式生成的草稿按模板结构填充，客户可在此基础上修改。
- **订单属性**：
  - **订单性质**（私密订单/公开订单）：不在订单广场公开展示（私密）或在订单广场可展示（公开）；接单方式见下。
  - 其他**订单属性**（如预算区间、工作量等）与发布模板字段一致或扩展，用于订单展示与订单广场筛选，具体在需求文档中定义。
  - **私密订单**：接单方式可为**指定接单方**（选择平台内某用户）或**指定由平台运营方接单**。
  - **公开订单**：接单方式可为**指定接单方**（仅被指定用户可见/可接）或**公开抢单**（在订单广场展示，供所有用户报名/竞标）。
- **订单状态与流转**：订单状态包括**新建**、**待审核**、**已打回**、**推广中**（含未上架/已上架）、**协议签署中**、**交付中**、**已验收**、**待结算**、**已结算**、**已取消**、**已终止**。提交后由运营方审核；审核通过 → **推广中**，不通过 → **已打回**（客户修改后可再提交 → 待审核）。**待审核**、**推广中**可由客户或运营方取消 → **已取消**；取消为逻辑删除，报名记录保留。选标/确认接单后 → **协议签署中**；经**客户上传 → 接单方确认 → 平台审核通过**使协议版本**生效**并关联订单 → **交付中**。**交付中**若某里程碑超期未达成则开放客户终止权 → **已终止**；终止后结算以双方协议为准，由运营人工审核并按条款执行，平台不自动结算。指定接单方在响应时限内未响应或拒绝时，按配置自动下架或转为公开抢单；公开抢单招标时限超时后按配置自动下架或自动隐藏。全部里程碑通过 → **已验收** → **待结算** → **已结算**。详见第 4 章「订单状态流转」。
- **审核操作**：运营方进行通过/打回，打回时需填写原因，客户可见。审核通过后，系统按订单属性与接单方式处理：公开抢单的订单上架订单广场；指定接单方的订单仅对指定用户开放；运营方接单的订单由运营方在后台承接。
- **详见后续《客户发布订单与审核》需求文档。**

### 6.4 订单广场与用户筛选、报名/竞标

#### 6.4.1 简述
**公开且选择公开抢单**的订单在订单广场展示，用户筛选、报名/竞标，客户选标；**指定接单方**的订单仅对被指定用户可见/可接；**私密且运营方接单**的订单由运营方在后台承接。

#### 6.4.2 详情 / 关联文档
- **订单广场**：为**客户与用户统一入口**，所有账户从该入口进入【订单广场】页面；需先完成网站账号登录。网站支持 **CSDN App 扫码快速创建账号并绑定 CSDN 账户**，扫码完成后可直接进入站内流程。**首次进入**订单广场时，若该账号**尚未选择身份**，则需**选择身份**（**客户**或**用户**，二选一），选择后进行**对应资质认证**（客户为企业资质认证，用户为个人/团队能力与资质），认证完成后再次进入订单广场**不再弹出**身份选择或认证页面。**页面展示**：客户与用户进入后页面内容**大致相同**，均包含**完整订单列表**，点击具体订单可**查看订单详情**。**客户**：可见【发布订单】按钮并可进行**发单**操作，**不可接单**；**用户**：不可见【发布订单】按钮，可**接单**（报名/竞标）。订单广场仅展示**公开 + 公开抢单**的订单；列表支持排序与分页。
- **分类与搜索**：用户可按以下维度对订单进行**分类和搜索**（具体筛选项与交互在需求文档中定义）：
  - **订单类别**：按**任务类型**分类（文本创作、视频创作、软件开发、产品评测、营销推广、平面设计、**硬件订单**）。
  - **订单属性**：按**订单属性**筛选（含**订单性质**如私密/公开、**预算区间**、**工作量**等，与发布模板字段一致或扩展）。
  - **交付模式**：按客户发布订单时填写的**订单交付模式**筛选。
  - **是否无人接单**：筛选「推广中且尚未确定接单方」的订单（无人接单）或已有人报名的订单。
  - **新订单**：按发布时间排序或标记为新订单（如最近 N 天内发布），便于用户发现最新订单。
  - 支持关键词搜索（如订单标题、描述等），具体范围在需求文档中定义。
- **报名/竞标**：用户对某订单「报名」或「竞标」（命名以产品最终确认为准），支持多条报名记录；客户在订单详情或管理端查看报名列表。
- **关注订单**：用户可**关注**订单广场上已发布的订单，关注后可在**【我的订单】**页面的「已关注的订单」页签中集中查看；支持取消关注。关注与报名相互独立（可仅关注不报名，或先关注后报名）。
- **选标**：客户选择一名用户承接该订单，选定后订单状态由**推广中**流转为**协议签署中**；经**客户上传 → 接单方确认 → 平台审核通过**使**订单三方协议**版本**生效**并关联订单后，流转为**交付中**，进入里程碑确认环节。
- **指定接单方**：客户在创建订单时指定某用户为接单方时，该订单仅对该用户可见（如「指定给我的订单」入口）；该用户可在**响应时限**内确认接单或拒绝。**确认接单**后订单状态由**推广中**流转为**协议签署中**；**订单三方协议**版本按「上传—确认—审核」**生效**后流转为**交付中**，直接进入里程碑确认。**拒绝**或**时限内未响应**时，按订单配置执行**自动下架**或**自动转为公开抢单**（转为公开后按招标时限在订单广场展示，供用户报名/竞标）。
- **公开抢单与招标时限**：公开抢单订单在订单广场展示，受**招标时限**约束；超时后按配置**自动下架**或**超时订单自动隐藏**（不再在订单广场展示），具体在需求文档中定义。
- **平台运营方接单**：私密订单且选择「由平台运营方接单」时，订单由运营方在后台承接，不向普通用户展示；运营方确认接单后，订单状态由**推广中**流转为**协议签署中**，**订单三方协议**版本按「上传—确认—审核」**生效**后流转为**交付中**，后续里程碑与结算流程与普通订单一致（接单方为运营方身份）。
- **详见后续《订单广场与报名竞标》需求文档。**

### 6.5 里程碑确认与交付验收

#### 6.5.1 简述
客户与用户共同确认订单的交付里程碑与总工期（支持自定义）；在各里程碑节点用户提交交付物、客户确认；全部里程碑通过后订单正式结算。

#### 6.5.2 详情 / 关联文档
- **交付中前置条件**：订单进入**交付中**的必要前置条件为**订单三方协议**版本在平台经「上传—确认—审核」**生效**并**自动关联当前订单**；选标/确认接单后先进入**协议签署中**，完成上述生效流程后方可进入交付中。
- **里程碑定义**：根据订单的**结算模式**，由**客户**（平台抽佣）或**平台运营方**（平台承包转包）与用户共同确认里程碑与总工期。支持自定义数量与内容，如总工期 3 个月、每月一个里程碑及对应交付说明（如 Month1 完成 ABC、Month2 完成 DEF、Month3 整体验收）。
- **节点流程**：用户在里程碑节点提交交付物（或交付说明）→ 客户确认；确认通过则该里程碑通过，未通过可要求修改后再次提交。
- **里程碑超期与客户终止权**：客户与用户商定的某个里程碑超过约定期限（如约定完成日 + 宽限期，具体天数或配置方式在需求文档中定义）未达成时，**开放客户的终止订单权利**（此前不开放）。客户行使终止权后订单进入**已终止**。**终止后结算**以客户与用户的相关协议为准，由**运营端人工审核**双方提供的合同/协议内容并按条款执行结算；**平台不承诺系统自动结算**，以规避平台法律责任。具体审核流程与结算执行在需求或后续迭代中细化。
- **协议与结算原则**：选标/确认接单后进入**协议签署中**，就**订单三方协议**（客户、用户、平台）在平台完成**上传—确认—审核**并使协议版本**生效**；原则上**用户与 CSDN 结算**。
- **结算与状态**：全部里程碑通过后，订单状态由**交付中**流转为**已验收**，再进入**待结算**；结算流程完成后流转为**已结算**（具体结算动作与支付在需求或后续迭代中定义）。
- **详见后续《里程碑确认与交付验收》需求文档。**

### 6.6 开发工具配置与订单资源发放、用量查看（火山云侧）

#### 6.6.1 简述
运营端配置订单的**可选工具列表**或必选工具；**接单方（用户）**可在**【我的订单】-【交付中的订单】**中进入订单详情，**配置该订单采用的交付工具**（从可选列表中**选定一个或多个工具**，如 Cursor + Trae 协同交付）；交付工具分为文档工具、文档创作工具、原型设计工具、音视频生成工具、开发工具等类别。**资源在订单三方协议版本于平台生效后发放**，订单**已结算或已取消/已终止**后资源**自动失效**；交付中可切换交付工具，切换后旧资源优先失效（做不到则保持可用）。工具与资源遵循既定采购与来源规则。订单系统本期不建设用量聚合页面，运营在火山云平台查看资源用量。

#### 6.6.2 详情 / 关联文档
- **运营端配置**：按任务类型或按订单配置「**可选工具列表**」或「必选某一/多个工具」；与订单发布模板或订单属性关联（在需求文档中明确配置层级与优先级）。运营控制的是「该订单可从哪些工具中选」及是否允许多选。
- **用户端配置交付工具**：
  - **入口**：用户在**【我的订单】-【交付中的订单】**下看到每个订单，**点击查看订单详情**后，可**配置此订单采用的交付工具**，从运营为该订单配置的**可选工具列表**中**选择一个或多个工具**（如软件开发订单可选 Cursor + Trae 协同交付）；若运营配置为必选某一工具，则需包含该工具，并可再选其他工具。
  - **多工具**：同一订单支持**多选多个工具**。订单系统记录工具配置与切换过程，用量查看由火山云平台侧提供。
- **交付工具分类**：支持选择的交付工具包括但不限于以下类别，具体工具枚举与分类在需求文档中定义：
  - **文档工具**
  - **文档创作工具**
  - **原型设计工具**
  - **音视频生成工具**
  - **开发工具**
  - 其他类别可按业务扩展。
- **工具与资源的采购与来源规则**（业务规则说明，具体采购流程、渠道码、授信与团购细节在需求或商务文档中细化）：
  - **部署的云**：须为 **CSDN 指定**的云，由 CSDN 购买或委托 OPC 采购（CSDN 提供渠道码）。
  - **开发工具**（如 TRAE 企业版等）：原则上由 **CSDN 指定购买**（CSDN 提供渠道码、授信）。
  - **设计工具、文档工具、音视频制作工具**：可根据**创作者使用习惯**购买；可参与**数字游民基地**由 CSDN 组织 OPC 的「**团购**」等方式获取。
- **资源发放时机**：在**订单三方协议版本于平台生效**（订单进入**交付中**）之后发放；用户在交付中的订单详情或「开发工具/资源」入口查看并获取当前订单绑定的 API Key、激活码等（具体形态以现有或规划中的资源发放能力为准）。
- **资源失效与回收**：订单进入**已结算**或**已取消**（含**已终止**）后，该订单绑定的资源**自动失效/收回**。
- **交付中切换交付工具**：同一订单在交付中支持**切换或增删**所选交付工具。切换后**之前工具的资源优先失效**；若下游系统无法做到随时控制资源失效，则**保持可用**。用量查看由火山云平台侧提供，订单系统不承载按订单或按工具的聚合查询页面。
- **用量查看（火山云侧）**：开发工具资源用量（如 API 调用量、token 等）由火山云平台统计与展示；运营需要查看时直接登录火山云控制台查询，订单系统内不提供集中查询入口。
- **详见后续《开发工具配置与订单资源用量》需求文档。**

### 6.7 客户/用户「我的订单」与运营订单管理

#### 6.7.1 简述
**客户**拥有**订单看板**与**订单管理系统**，可查看本人所发出的**全部订单**信息及状态（含**已取消、已终止**），支持**「仅看有效订单」**过滤，并按状态/类型/时间筛选，进入订单详情后对用户报名、选标、协议签署、里程碑确认、终止订单（当里程碑超期开放时）进行操作；拥有**【我的账单】**入口。**用户**通过**【我的订单】**入口进入「我的订单」页面，页面内包含**指定给我的订单**（指定给本人且尚未确认接单的订单）、**已关注的订单**、**已接订单**、**交付中的订单**、**已验收订单**（待结算/已结算）等页签，确认接单后订单自「指定给我的订单」归入已接订单等；支持按状态筛选时包含已取消、已终止；拥有**【我的账单】**入口。**运营端**统一查看与筛选全部订单，配合审核与配置使用。

#### 6.7.2 详情 / 关联文档
- **客户订单看板与订单管理**：
  - **订单看板**：客户端集中展示本人所发全部订单的看板或列表视图，支持按**订单状态**、任务类型、时间等筛选，**确保客户能查看所发出的所有订单信息**（含状态、类型、报价、接单方式、接单方、里程碑进度等）。**已取消、已终止**订单在列表中展示；需提供**「仅看有效订单」**过滤按钮，便于默认或按需排除已取消、已终止订单。看板按状态显示订单时，须支持**已取消**、**已终止**状态的展示与筛选。
  - **订单管理系统（客户侧）**：在看板/列表基础上，提供订单详情、报名列表（公开抢单时）、选标、指定接单方管理、**协议签署**（或等待协议签署）、里程碑确认、**终止订单**（当里程碑超期开放时）等操作入口；客户可从看板进入任一订单进行后续操作。
  - 看板视图形式（如列表/卡片/状态分栏等）、默认排序与筛选项在需求文档中定义。
- **客户【我的账单】入口与页面**：
  - **入口**：客户需拥有**【我的账单】**入口（如主导航、「我的」等），点击后进入「我的账单」页面。
  - **我的账单页面（客户侧）**：展示**本人所发出的订单相关的各种账单信息**；支持按订单、时间、账单类型等筛选与查看；账单与订单关联，客户可查看某订单下的应付、支付状态、结算状态等明细，具体展示项、账单类型与业务规则在需求文档中定义。
- **客户**：我发布的订单列表（全部订单）、订单看板、订单详情、报名列表（公开抢单时）、选标、指定接单方管理、**协议签署**、里程碑确认、**终止订单**（里程碑超期开放时）、**【我的账单】**入口与我的账单页面。
- **用户【我的订单】入口与页面**：
  - **入口**：页面中新增**【我的订单】**入口（如主导航、「我的」、订单广场等），用户点击后进入「我的订单」页面。
  - **我的订单页面**：以页签形式组织，各页签下展示**当前用户相关**的订单数据；页签至少包含：
    - **指定给我的订单**：展示客户**指定给当前用户、尚未确认接单**的订单（订单处于推广中、指定接单方为本人）；用户可在此页签内确认接单或拒绝。**确认接单后**该订单自本页签移除，**统一归入「已接订单」**，并随订单状态进入协议签署中、交付中的订单等；拒绝或超时未响应时按订单配置下架或转为公开抢单。每条订单提供**【沟通】按钮**，点击后弹出沟通信息弹框，展示发单客户联系方式，便于接单前沟通细节。
    - **已关注的订单**：当前用户已关注、且尚未承接的订单（来自订单广场的关注列表）；可取消关注、可跳转报名。
    - **已接订单**：当前用户已承接的订单（含客户选标确定本人、或指定接单方确认接单的订单），不限状态；可进入订单详情、里程碑提交与进度查看、开发资源获取等。
    - **交付中的订单**：当前用户已承接且订单状态为**交付中**的订单（里程碑进行中）；需能够展示**当前订单的状态**与**里程碑信息**，便于用户了解进度并快速进入里程碑提交与确认；**点击查看订单详情**后，可**配置该订单采用的交付工具**（从运营配置的可选工具列表中**选定一个或多个工具**）；具体展示形式（如列表字段、卡片信息、进度条等）在需求文档中定义。
    - **已验收订单**：当前用户已承接且全部里程碑已通过的订单（订单状态为已验收、待结算或已结算）；该页签下分两部分展示：**待结算订单**（已验收、等待与 CSDN 结算）、**已结算订单**（与 CSDN 结算已完成）；可进入订单详情、查看结算与账单相关信息，具体展示形式在需求文档中定义。
  - 各页签下均展示与当前用户相关的订单列表，支持进入订单详情；**交付中的订单**页签下需展示订单状态与里程碑信息，具体展示形式在需求文档中定义。页签可依需求扩展（如「我报名的订单」等），具体在需求文档中定义。
- **用户【我的账单】入口与页面**：
  - **入口**：用户需拥有**【我的账单】**入口（如主导航、「我的」等），点击后进入「我的账单」页面。
  - **我的账单页面**：展示**本人参与每一个订单所产生的各种账单信息**；支持按订单、时间、账单类型等筛选与查看；账单与订单关联，用户可查看某订单下的收入、扣款、结算状态等明细，具体展示项、账单类型与业务规则在需求文档中定义。
- **用户**：**【我的订单】**入口、我的订单页面（**指定给我的订单**、已关注的订单、已接订单、**协议签署中**、交付中的订单、**已验收订单**（待结算订单、已结算订单）及已取消/已终止的展示或筛选）、**【我的账单】**入口与我的账单页面、协议签署、里程碑提交与进度查看、开发资源获取入口。用户侧订单列表支持按状态筛选时包含**已取消、已终止**（如已接订单中历史订单可按状态筛出），具体在需求文档中定义。
- **运营**：**业务看板**（可查看当前平台客户、用户、订单、账单与结算、合同、后台设置等）、订单列表（按**订单状态**、类型、时间等筛选，含协议签署中、已终止等）、**待运营方接单的私密订单**（可承接）、订单详情、审核与配置入口、终止订单结算审核（按双方协议人工执行）；资源用量在火山云平台查看。

## 9. 风险与假设

### 9.1 关键假设

- 订单广场为**统一入口**，用户需先通过网站账号登录后进入；网站支持 **CSDN App 扫码快速创建账号并绑定 CSDN 账户与网站账户**。**首次进入**订单广场时，若该账号尚未选择身份，则需**选择身份**（**客户**或**用户**，二选一），选择后完成**对应资质认证**，认证完成后再次进入不再弹出身份选择或认证页面。**单一身份**：同一账号仅能具备客户或用户身份之一，**客户**仅能发单、不可接单，**用户**仅能接单、不可发单。**仅运营人员**可同时以“发单”与“承接”角色存在（如创建模版创建订单并自行承接），不视为普通账号的单一身份逻辑。若业务后续需支持双身份，需在需求中明确并扩展。
- 运营方使用 CSDN 既有运营/管理员账号与权限，不新增独立注册流程。
- 开发工具与订单的配置粒度（按任务类型、按订单、或两者结合）及与现有代理系统的对接方式，以需求文档与技术方案确认为准。

### 9.2 主要风险与应对

- **企业认证与用户资料标准不统一**：认证项或资料字段变更会影响注册与展示。应对：在需求阶段与业务确认认证项与资料结构，尽量在开发前冻结核心字段。
- **代理工具对接与计量口径不一致**：不同工具 API、计量维度不同，若后续回收用量聚合到订单系统会增加对接复杂度。应对：本期先收敛为资源绑定与发放/失效闭环，用量查看统一走火山云平台，后续再评估聚合。

### 9.3 外部依赖

- **账号与权限**：依赖独立网站账号体系、登录能力与运营/管理员权限配置；需支持网站独立注册/登录，以及 **CSDN App 扫码快速建号 + 账户绑定**流程；若需新增「订单审核」等角色，依赖权限系统扩展。
- **开发工具与资源**：依赖 CSDN 代理工具的发放接口与火山云平台用量查看能力；若暂无「按订单绑定」能力，需与相关系统共建或在本系统内实现最小闭环。
- **客户认证材料审核**：客户企业资质认证依赖营业执照与资质证书等材料的提交质量与审核能力；若材料缺失、模糊、过期或信息不一致，需在需求中约定补件、驳回与再次提交规则。
- **订单创建 AI 解析**：「上传文档由 AI 解析创建」与「自然语言 AI 解析创建」依赖后台大模型/LLM 能力；解析输出需与订单模板字段对齐，若解析失败或质量不足，需在需求中约定降级策略（如提示客户改用手工创建或重新输入）。

| 新建 | Draft | 订单状态之一：客户创建订单草稿，未提交审核；可编辑、可删除、可提交。 |
| 待审核 | Pending Review | 订单状态之一：客户已提交，等待运营方审核；运营方可「通过」或「打回」。客户与运营方均可在此阶段取消订单。 |
| 已打回 | Rejected | 订单状态之一：运营方审核不通过；客户可修改后重新提交，提交后 → 待审核。 |
| 推广中 | Promoting | 订单状态之一：运营方审核通过，订单已发布，等待确定接单方；区分**未上架**/ **已上架**（如指定接单方尚未确认为未上架，公开抢单已在订单广场展示为已上架）。客户与运营方均可在此阶段取消订单。指定接单方在响应时限内未响应或拒绝时，按配置自动下架或转为公开抢单；公开抢单招标时限超时后按配置自动下架或自动隐藏。 |
| 未上架 | Not Listed | 推广中订单的展示状态之一：订单已审核通过但尚未在订单广场或对指定方形成有效展示（如指定接单方尚未确认接单）。 |
| 已上架 | Listed | 推广中订单的展示状态之一：订单已在订单广场展示（公开抢单）或已对指定接单方可见并可操作。 |
| **协议签署中** | Agreement Pending | 订单状态之一：客户选标或指定接单方/运营方确认接单后进入（状态名保留；业务上为协议**确认与生效流程进行中**）。须完成**客户上传协议文件 → 接单方确认 → 平台运营审核通过**，使**订单三方协议**版本**生效**并**自动关联当前订单**；**协议版本生效**是进入**交付中**的必要前置条件。协议确认超时或一方拒绝时的处理在需求文档中定义。 |
| 响应时限 | Response Deadline | 指定接单方订单的必配项：被指定方需在该时限内确认接单或拒绝；超时未响应或拒绝时，按订单配置执行自动下架或自动转为公开抢单。支持常用时限预设或无限制；可在待审核、未上架、推广中（已上架）状态下由客户调整。 |
| 招标时限 | Bidding Deadline | 公开抢单订单的必配项：订单在订单广场接受报名的时长；超时后按配置自动下架或超时订单自动隐藏。支持常用时限预设或无限制；可在待审核、未上架、推广中（已上架）状态下由客户调整。 |
| 交付中 | In Delivery | 订单状态之一：**订单三方协议**版本已按「上传—确认—审核」**生效**并关联订单，里程碑进行中（尚未全部通过）；客户与接单方按里程碑提交与确认。某里程碑超过约定期限未达成时开放客户终止订单权利，订单可进入**已终止**。是否允许运营方在交付中取消/终止在需求文档中确认。 |
| 已验收 | Accepted | 订单状态之一：全部里程碑已通过，交付完成；可进入待结算。 |
| 待结算 | Pending Settlement | 订单状态之一：已验收，等待结算流程（支付、对账等）；具体规则在需求或后续迭代中定义。 |
| 已结算 | Settled | 订单状态之一：结算完成，订单闭环结束。 |
| 已取消 | Cancelled | 订单状态之一：客户或运营方在**待审核**、**推广中**阶段取消订单。取消采用逻辑删除，报名记录及相关数据保留在数据库中。与**已终止**区分：已取消为未进入或未完成交付的取消；已终止为交付中因里程碑超期等由客户行使终止权后的结束。 |
| **已终止** | Terminated | 订单状态之一：**交付中**阶段因某里程碑超过约定期限未达成，客户行使终止权（或运营方依规则操作）后订单结束。**终止后结算**以客户与用户的相关协议为准，由**运营端人工审核**双方提供的合同/协议内容并按条款执行结算；**平台不承诺系统自动结算**，以规避平台法律责任。 |
| 订单看板 | Order Kanban | 客户侧用于集中查看与管理**本人所发出的全部订单**的看板或列表视图；支持按订单状态、任务类型、时间等筛选，展示订单关键信息（状态、类型、报价、接单方式、里程碑进度等），并可进入订单详情进行报名列表、选标、里程碑确认等操作。 |
| 客户订单管理系统 | Customer Order Management | 客户侧订单管理能力统称，包含订单看板、全部订单列表、订单详情及报名/选标/里程碑确认等操作；确保客户能够查看并管理所发出的所有订单信息。 |
| 私密订单 | Private Order | **订单性质**的一种：客户创建订单时设置为私密；不在订单广场公开展示。接单方式可为**指定接单方**或**指定由平台运营方接单**。 |
| 公开订单 | Public Order | **订单性质**的一种：客户创建订单时设置为公开；接单方式可为**指定接单方**或**公开抢单**。仅当选择公开抢单时，订单在订单广场展示。 |
| 指定接单方 | Assigned Contractor | 客户在创建订单时指定某用户（或运营方）为接单方；该订单仅对被指定方可见/可接，不进入订单广场公开展示。 |
| 平台运营方接单 | Operation-assigned Order | 私密订单的一种接单方式：客户指定由平台运营方承接该订单；运营方在后台承接后，后续里程碑与结算流程与普通订单一致。 |
| 公开抢单 | Open Bidding | 公开订单的一种接单方式：订单在订单广场公开展示，供用户报名/竞标，客户从报名用户中选标确定接单方。 |
| 订单广场 | Order Plaza | 已发布且**公开 + 公开抢单**订单的集中展示与筛选入口，为**客户与用户统一入口**；所有账户从该入口进入【订单广场】页面，需先完成网站账号登录。网站支持通过 **CSDN App 扫码**快速创建网站账号并绑定 CSDN 账户，扫码完成后可直接进入站内流程。**首次进入**订单广场时需**选择身份**（客户或用户二选一），选择后完成**对应资质认证**，认证完成后再次进入不再弹出身份选择或认证页面。**页面展示**：客户与用户进入后内容大致相同，均包含完整订单列表、点击订单可查看订单详情；**客户**可见【发布订单】按钮并可发单、**不可接单**，**用户**不可见【发布订单】、可接单。支持按**订单类别**（任务类型）、**订单属性**（含订单性质、预算区间、工作量等）、**交付模式**、**是否无人接单**、**新订单**等维度分类和搜索，具体筛选项在需求文档中定义。 |
| 关注订单 | Follow Order | 用户对订单广场上已发布的订单进行关注的操作；关注后的订单出现在用户侧**【我的订单】**页面的「已关注的订单」页签中，便于后续查看与报名；支持取消关注。 |
| 我的订单（用户侧） | My Orders (Contractor) | 用户侧入口与页面名称：**【我的订单】**入口，进入后为「我的订单」页面；页面以页签形式组织，包含**指定给我的订单**（指定给本人、尚未确认接单的订单）、**已关注的订单**、**已接订单**、**交付中的订单**、**已验收订单**（下含**待结算订单**、**已结算订单**）等页签。指定给我的订单确认接单后归入已接订单；该页签订单项提供**【沟通】按钮**，点击后弹框展示发单客户联系方式；支持按状态筛选时包含已取消、已终止。 |
| 指定给我的订单 | Assigned-to-Me Orders | 用户【我的订单】页面下的页签之一：展示客户**指定给当前用户、尚未确认接单**的订单（推广中、指定接单方为本人）；用户可在此确认接单或拒绝。确认接单后订单归入「已接订单」，并随状态进入协议签署中、交付中等。 |
| 已验收订单 | Accepted Orders Tab | 用户【我的订单】页面下的页签之一：展示当前用户已承接且全部里程碑已通过的订单；该页签下分**待结算订单**（已验收、等待与 CSDN 结算）与**已结算订单**（与 CSDN 结算已完成）两部分，可进入订单详情、查看结算与账单相关信息。 |
| 待结算订单 | Orders Pending Settlement | 用户【我的订单】-【已验收订单】页签下的分类之一：已验收、尚未与 CSDN 完成结算的订单列表。 |
| 已结算订单 | Settled Orders | 用户【我的订单】-【已验收订单】页签下的分类之一：与 CSDN 结算已完成的订单列表。 |
| 我的账单（用户侧） | My Bills (Contractor) | 用户侧入口与页面名称：用户通过**【我的账单】**入口进入「我的账单」页面，可查看**本人参与每一个订单所产生的各种账单信息**（一订单可有多笔账单，如按阶段结算）；支持按订单、时间、账单类型等筛选，账单与订单关联；当订单下全部应结账单已支付/已入账后，订单可进入已结算。具体展示项与业务规则在需求文档中定义。 |
| 我的账单（客户侧） | My Bills (Customer) | 客户侧入口与页面名称：客户通过**【我的账单】**入口进入「我的账单」页面，可查看**本人所发出的订单相关的各种账单信息**（一订单可有多笔账单，如按阶段结算）；支持按订单、时间、账单类型等筛选，账单与订单关联，具体展示项与业务规则在需求文档中定义。 |
| 账单 | Bill / Invoice | 与订单关联的结算单据，独立对象；一订单可关联多笔账单（按阶段结算时每阶段/里程碑可对应一笔）。账单具备自身状态流转（待确认、待支付、支付中、已支付、已入账、已冲销、已关闭等），与订单状态协同。订单进入待结算时可自动创建账单；按阶段结算时交付中即可按里程碑达成生成阶段账单，待结算时生成尾款账单。 |
| 账单状态 | Bill Status | 账单在生命周期内的状态，**单独定义**，与订单状态协同（关系见第 4 章「账单与结算」）。建议包含：**待确认**（已生成、待确认金额与条款）、**待支付**（已确认、等待支付）、**支付中**、**已支付**、**已入账**（结算完成）、**已冲销**（作废）、**已关闭**（取消结算）。具体枚举与流转在需求文档中定义。 |
| 按阶段结算 | Phase-based Settlement | 订单可配置的结算方式：按里程碑/阶段分别生成账单并独立结算；交付中每达成一里程碑可生成该阶段账单，订单全部交付完成进入待结算时再生成尾款账单。支持部分结算+剩余取消（如仅结算已达成里程碑对应账单）。 |
| 结算动作 | Settlement Action | 将账单从待确认/待支付推进至已支付、已入账的操作；不强制绑定在里程碑完成时刻，支持客户、用户或运营在账单就绪后按权限灵活发起。 |
| 业务看板（运营端） | Operation Dashboard / Business Kanban | 运营端集中入口与总览页面：运营人员可在此查看当前平台的**客户**、**用户**、**订单**、**账单与结算**、**合同**、**后台设置**等核心内容，并从看板进入各模块列表或配置页；具体看板布局、统计维度与跳转逻辑在需求文档中定义。 |
| **订单三方协议** | Order Tripartite Agreement | 客户、用户、平台（CSDN）为权利义务主体的协议文本。在系统中通过**客户上传协议文件 → 接单方确认 → 平台运营审核通过**使协议版本**生效**并**自动关联订单**，作为进入**交付中**的必要前置条件；**非**平台内第三方电子签或「在线缔约」产品。协议内容来源、模板与上传规则见《订单广场与报名竞标》等需求文档；若后续引入电子签专项，在独立需求中约定。 |
| 用户与CSDN结算 | User–CSDN Settlement | 结算关系原则：结算主体为用户—CSDN，即用户与平台结算；客户与平台、平台与用户的资金与结算规则在需求或后续迭代中补充。**已终止**订单的结算以客户与用户协议为准，由运营人工审核并按条款执行，平台不自动结算。 |
| 报名/竞标 | Bid / Application | 用户对某一订单表示兴趣并参与竞争的行为；多用户可对同一订单报名，由客户选定一名承接。 |
| 里程碑 | Milestone | 客户与用户共同确认的交付节点，包含节点顺序、工期与交付内容；用户提交、客户确认通过后该节点通过。 |
| 开发工具/资源 | Dev Tools / Resources | 与订单绑定的 API Key、激活码等开发或 AI 工具资源。**发放时机**：**订单三方协议**版本于平台**生效**（订单进入交付中）之后；**失效**：订单已结算或已取消/已终止后自动失效。同一订单支持多选多个交付工具；交付中可切换工具，切换后旧资源优先失效（做不到则保持可用）。本期订单系统不建设用量聚合统计页面，资源用量查看由火山云平台提供。 |
| 交付工具 | Delivery Tool | 接单方（用户）在执行某订单交付时选用的工具；运营为订单配置「可选工具列表」或「必选某一/多个工具」，用户可在【我的订单】-【交付中的订单】订单详情中从可选列表中**选定一个或多个工具**（如 Cursor + Trae 协同交付）；各工具**分别统计用量**。分类包括文档工具、文档创作工具、原型设计工具、音视频生成工具、开发工具等。 |
| 交付工具分类 | Delivery Tool Category | 交付工具按用途划分的类别：文档工具、文档创作工具、原型设计工具、音视频生成工具、开发工具等；具体枚举在需求文档中定义。 |
| 工具与资源采购规则 | Tool & Resource Procurement | 业务规则：部署的云须 CSDN 指定（CSDN 购买或委托 OPC 采购，渠道码）；开发工具原则上 CSDN 指定购买（渠道码、授信）；设计/文档/音视频工具可按创作者习惯购买，可参与数字游民基地 CSDN 组织 OPC 的团购等。 |
| 任务类型 | Task Type | 订单的分类维度，**固定 8 种**：文本创作、视频创作、软件开发、产品评测、营销推广、平面设计、**硬件订单**、模版创建。其中**模版创建**仅运营端可创建，客户端不可选。由运营方在后台配置，并关联发布模板。 |
| 模版创建 | Template Creation | 任务类型之一，**内部自用任务**：仅运营端可创建该类型订单；**不需要用户承接**，仅支持运营端承接；**不走里程碑与结算**，该两步直接跳过，采用**简化流程**（如审核通过后运营方承接即视为完成），具体状态流转在需求文档中定义。 |
| 订单创建模式 | Order Creation Mode | 创建订单时的三种方式，每种任务类型均支持，且可与订单模板结合使用：**手工创建**（按表单项逐项填写）、**上传文档由 AI 解析创建**（上传文档后 AI 生成订单草稿）、**自然语言 AI 解析创建**（用户用自然语言描述需求，AI 生成订单草稿）。 |
| 手工创建 | Manual Creation | 订单创建模式之一：客户按订单模板或默认表单项逐项手动填写订单内容。 |
| 上传文档由 AI 解析创建 | Document Upload + AI Parsing | 订单创建模式之一：客户上传文档（如 Word、PDF），后台 AI 解析文档内容并生成订单草稿，客户确认或修改后提交。 |
| 自然语言 AI 解析创建 | Natural Language + AI Parsing | 订单创建模式之一：客户用自然语言描述需求，后台 AI 解析并生成订单草稿，客户确认或修改后提交。 |
| 发布模板 | Publish Template | 与任务类型对应的订单填写模板，定义客户需填写的字段与校验规则，由运营方在后台配置；可与三种订单创建模式结合使用。 |
