# 消息通知情况总结

本文档基于当前仓库代码梳理：**谁在什么时机发通知、通过哪些渠道、站内消息是否落库**。统计口径截至代码现状（以 `CompositeNotificationService`、`AuthAppService`、`MessageController` 等为准）。

---

## 1. 通知渠道概览


| 渠道       | 实现要点                                                                                                  | 说明                                                        |
| -------- | ----------------------------------------------------------------------------------------------------- | --------------------------------------------------------- |
| **站内消息** | `CompositeNotificationService.persistNotification` → `NotificationMapper` 写入通知表                       | 前端通过 `/api/v1/messages*` 拉列表、角标、标记已读（`MessageAppService`） |
| **短信**   | `SmsService` → 默认 `**VolcSmsServiceImpl`**（`@Primary`）；AK 未配或缺失模板时走 **Mock 日志**                       | 业务通知与验证码共用同一套服务接口                                         |
| **邮件**   | `EmailService` → `**AliyunEmailServiceImpl`**（`@Primary`，实为 SMTP/`MailProperties`）；配置不完整时 **Mock 日志** | 通知正文走 `sendEmail(subject, content)`                       |


补充：`SmsServiceImpl`、`EmailServiceImpl` 为条件 Bean（缺省时兜底），与当前 `**VolcSmsServiceImpl` + `AliyunEmailServiceImpl`** 并存时需以 `@Primary` 实际注入为准。

---

## 2. 认证类通知（与订单等业务分离）

由 `AuthAppService` + `SmsClient` / `EmailClient` 触发，走 `**SmsService.sendVerificationCode` / `EmailService.sendVerificationCode**`，**不写站内通知表**。


| 场景       | 渠道           | 代码入口                                               |
| -------- | ------------ | -------------------------------------------------- |
| 注册验证码    | 短信           | `SmsClient.sendRegisterOtp`                        |
| 登录验证码    | 短信           | `SmsClient.sendLoginOtp`                           |
| 重置密码验证码  | 短信（复用登录 OTP） | `SmsClient.sendLoginOtp`                           |
| 换绑手机号验证码 | 短信           | `SmsClient.sendChangePhoneOtp`（模板待短信平台确认，见代码 todo） |
| 绑定邮箱验证码  | 邮件           | `EmailClient.sendBindEmailOtp`（模板待邮件平台确认，见代码 todo） |


开发环境下可开启 OTP Mock，验证码仅打日志返回。

## 3. 业务通知清单（`NotificationService` → `CompositeNotificationService`）

下列场景在实现中通常为：**有手机号则发短信、有邮箱则发邮件**；表中「站内」表示是否调用 `persistNotification`。
