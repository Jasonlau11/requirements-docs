# TaoToken 参会端显式勾选修订产品需求说明书

## 需求概览

对 `TaoToken引流集成产品需求说明书.md` 中**参会人侧**的注册与额度发放逻辑做实质性修订：从"办会人统一决策、参会人无感知静默注册发放"改为"办会人开启激励后，参会人**主动勾选同意**才注册 TaoToken 账号并领取额度"。不勾选的参会人不受影响，正常完成报名/签到流程但不涉及 TaoToken。同步新增外链报名场景下通过**技术会议平台小程序**实现签到并发放 3.99 元券的能力（小程序待评估）。

### 1.3 背景和价值

**背景问题**：原 TaoToken 文档 V1.0 中，参会人报名时由后台自动以手机号静默注册 TaoToken 账号并发放额度，参会人无感知、无选择权。这种模式存在两个核心问题：

1. **合规风险**：在未获得参会人明确同意的情况下注册第三方平台账号并发放额度，与个保法"告知-同意"原则冲突。
2. **骚扰感知**：部分参会人不希望被注册额外账号或收到 TaoToken 相关推送，静默模式导致体验劣化。

**业务价值**：改为显式勾选后，参会人对 TaoToken 注册拥有完全自主权；仅真正有意向的用户进入 TaoToken 体系，提升后续活跃转化质量。同时平台满足合规要求。

**用户价值**：参会人可自主决定是否注册 TaoToken；不勾选的用户报名和签到体验不受任何影响。

## 第2章：功能需求

### 2.1 参会端 TaoToken 勾选框（TT-Revised-01）

#### 场景描述

参会人在会议详情页点击"报名"进入报名表单区域。如果该会议已由办会人开启 TaoToken 激励，则报名按钮下方的协议区域新增一个 TaoToken 勾选框，作为独立第三项。勾选默认选中。参会人可主动取消勾选——取消后，报名和签到均不触发 TaoToken 注册与额度发放，其余流程不受影响。若办会人未开启 TaoToken 激励，该勾选框不展示。

#### 用户故事

| 编号 | 用户故事 |
|------|----------|
| US-TT-REV-01 | 作为参会人，我希望在报名时看到 TaoToken 注册选项（独立勾选框），以便自主决定是否注册 TaoToken 账号并领取额度 |
| US-TT-REV-02 | 作为参会人，我希望 TaoToken 勾选框默认选中（与办会人意愿一致），以便便利地享受赠送福利 |
| US-TT-REV-03 | 作为参会人，我希望取消勾选后报名流程不受任何影响，以便不被强制注册第三方账号 |
| US-TT-REV-04 | 作为不关心 TaoToken 的参会人，我希望在办会人未开启 TaoToken 的会议中不看到任何 TaoToken 相关内容，以便报名表单保持简洁 |

#### 需求规格

**2.1.1 勾选框展示规则**

| 规则项 | 内容 |
|--------|------|
| 展示条件 | 仅当当前会议的办会人开启了"报名赠送 1.99 元 TaoToken 额度"选项时才展示 |
| 不展示条件 | 办会人未开启 TaoToken 激励时，此项不出现在报名表单中 |
| 展示位置 | 报名按钮下方，现有两项协议之后，作为独立的第三项 |
| 排列顺序 | 1. 阅读并同意《用户协议》和《隐私政策》（两项一起） → 2. **勾选即代表同意注册 CSDN TaoToken 账号，完成报名可领取 1.99 元 Token 抵扣券** |
| 默认状态 | **默认勾选** |
| 可编辑性 | 参会人可主动取消勾选 |

**2.1.2 勾选文案**

```
□ 勾选即代表同意注册 CSDN TaoToken 账号，完成报名可领取 1.99 元 Token 抵扣券
```

- 文案中的"CSDN TaoToken 账号"可点击，在新页签打开 TaoToken 介绍页或用户协议
- 若 TaoToken 用户协议在文档交付时已就绪，则改为"勾选即代表同意《TaoToken 用户协议》，完成报名可领取 1.99 元 Token 抵扣券"，点击《TaoToken 用户协议》在新页签展示全文

### 2.2 报名注册与额度发放（TT-Revised-02）——覆盖原 2.3

#### 场景描述

参会人提交报名时，系统检查该参会人的 TaoToken 勾选状态。若勾选，则按异步队列模式调用 TaoToken 注册/发放接口；若不勾选，则跳过 TaoToken 所有逻辑。其余报名流程保持不变。

> **与 V1.0 的关键区别**：原逻辑中，所有参会人报名后自动触发 TaoToken 注册发放；新逻辑下，仅**勾选了 TaoToken 的参会人**才触发。

#### 用户故事

| 编号 | 用户故事 |
|------|----------|
| US-TT-REV-05 | 作为勾选了 TaoToken 的参会人，我报名成功后期望自动获得 TaoToken 额度，无需额外操作 |
| US-TT-REV-06 | 作为未勾选 TaoToken 的参会人，我报名成功后期望不产生任何与 TaoToken 系统的交互 |

#### 需求规格

**2.2.1 修订后的发放流程**

```
参会人填写报名信息 + TaoToken 勾选状态
  ↓
提交报名
  ↓
系统校验报名数据通过
  ↓
写入报名记录，状态 = 待审核
  ↓
检查：当前会议是否开启"报名赠送 TaoToken 额度"？
  ↓ 是
检查：参会人是否勾选了 TaoToken？
  ↓ 是（勾选）
将发放任务写入异步队列（phone, meeting_id, amount=1.99, trigger=registration）
  → 返回报名成功（不等待 TaoToken 接口返回）
  ↓
  ↓ 否（未勾选）
跳过 TaoToken 注册与发放 → 返回报名成功
  ↓
  ↓ 否（会议未开启）
跳过 → 返回报名成功

异步 Worker 消费队列（不变）：
  1. 调用 TaoToken 注册/查询接口（以手机号）
  2. 调用 TaoToken 额度发放接口（1.99 元）
  3. 成功后记录发放日志
  4. 失败后标记 retry_count，隔 5 分钟重试，最多 3 次
  5. 3 次均失败 → 记录为最终失败，触发告警
```

**2.2.2 修订后的规则**

| 规则项 | 内容 |
|--------|------|
| 账号关联键 | 手机号（不变） |
| 发放金额 | 1.99 元（不变） |
| 发放时机 | 报名提交成功即触发（不变） |
| 发放策略 | 异步队列，不阻断报名流程（不变） |
| 重复发放防护 | 同一手机号 + 同一会议 ID + 同一触发类型仅发放一次（不变） |
| 开关检查（修订） | **除检查会议级开关外，增加检查参会人的 TaoToken 勾选状态；两项均满足才触发发放** |
| 勾选状态存储 | 报名记录中新增 `taotoken_opted_in`（布尔）字段；报名提交时写入，不可修改 |
| 失败重试 | 间隔 5 分钟，最多 3 次（不变） |

**2.2.3 TaoToken API 接口（不变）**

沿用原 2.3.3 的接口约定，不修改。

#### 验收标准

| 编号 | 验收标准 |
|------|----------|
| AC-TT-REV-05 | 办会人开启了 TaoToken 激励 + 参会人勾选了 TaoToken → 报名成功后，后台触发 TaoToken 注册 + 1.99 元发放 |
| AC-TT-REV-06 | 办会人开启了 TaoToken 激励 + 参会人**未**勾选 TaoToken → 报名成功后，后台不触发任何 TaoToken 调用 |

### 2.3 签到奖励发放（TT-Revised-03）——覆盖原 2.4

#### 场景描述

勾选了 TaoToken 的参会人完成签到后，系统自动发放 3.99 元 TaoToken 额度。签到的触发方式有两种：
- **平台报名签到**：沿用现有扫码签到和手动签到逻辑。
- **外链报名签到**：通过**技术会议平台小程序**实现（新能力，待评估）。

未勾选 TaoToken 的参会人签到流程与上述一致，仅跳过 TaoToken 额度发放。

| US-TT-REV-08 | 作为通过外链报名且勾选了 TaoToken 的参会人，我希望现场通过小程序完成签到并领取 3.99 元额度 |
| US-TT-REV-09 | 作为未勾选 TaoToken 的参会人，我签到流程不受影响，不产生 TaoToken 相关操作 |

### 2.4 存量会议同步升级（TT-Revised-04）

#### 场景描述

本次修订发布后，**所有存量会议**（已发布且办会人开启了 TaoToken 激励的）须同步升级：参会人在存量会议的详情页报名时，同样显示 TaoToken 勾选框（默认勾选），按新逻辑执行。不允许两种逻辑（静默 vs 显式）在生产环境并存。

#### 需求规格

| 规则项 | 内容 |
|--------|------|
| 影响范围 | 所有已发布、办会人已开启 TaoToken 激励的会议 |
| 升级方式 | **数据库层面**：无需迁移存量报名记录；仅需在**前端**渲染报名表单时，按会议级 TaoToken 开关判断是否渲染第三项勾选框 |
| 存量报名记录 `taotoken_opted_in` | 旧记录此字段为 null。系统在该字段为 null 时不判断（不触发补发、不强制回填） |
| 存量报名记录签到 | 已签到的存量记录按**旧逻辑**生效（3.99 元已发放或已在队列中），不因字段 null 而被取消 |
| 新报名（发布后） | 所有新报名记录写入 `taotoken_opted_in`，按新逻辑执行 |

#### 验收标准

| 编号 | 验收标准 |
|------|----------|
| AC-TT-REV-16 | 存量会议（已开启 TaoToken）的详情页中，报名表单同步展示 TaoToken 勾选框 |
| AC-TT-REV-17 | 存量会议的已有报名记录不受影响，已发放的额度不追回 |
| AC-TT-REV-18 | 新报名记录正常写入 `taotoken_opted_in` 字段 |
