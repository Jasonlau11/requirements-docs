# TaoToken 引流集成产品需求说明书

## 需求概览

会议系统接入 CSDN 旗下 TaoToken 大模型额度中转站，通过报名赠送、签到奖励和详情页广告三种途径为其引流。办会人发布会议时可选择是否为参会人赠送报名额度（默认勾选）。参会人报名时，系统自动以其手机号静默注册/关联 TaoToken 账号并发放 1.99 元额度；完成签到后再发放 3.99 元额度。会议详情页底部新增 TaoToken 广告区域。

## 第1章：概述

### 1.1 术语表

| 名称 | 详细描述 |
|------|----------|
| TaoToken | CSDN 旗下大模型额度中转站系统，用户可通过消耗额度调用各大模型 API |
| 额度 | TaoToken 系统中的虚拟货币单位，1.99 元 / 3.99 元为赠送金额 |
| 静默注册 | 参会人报名时，会议系统在后台调用 TaoToken 注册接口，参会人无需感知注册过程 |
| 异步补发 | TaoToken 接口调用失败时，将发放任务写入重试队列，不阻塞报名流程，后续自动补发 |

### 1.2 修订记录

### 1.3 背景和价值

- **背景问题**：TaoToken 作为 CSDN 自研的大模型额度中转站，需要获取初始用户。会议系统的参会人（技术人群）与 TaoToken 的目标用户高度重叠，是天然的引流渠道。
- **业务价值（甲方视角）**：为 TaoToken 系统导入注册用户和活跃用户；会议场景为额度消费提供实际使用场景（参会者可能需要额度用于会议相关的 AI 工具）。
- **用户价值**：参会人免费获得 TaoToken 额度，降低试用门槛。

### 1.4 影响范围

| 端 | 影响 |
|----|------|
| 办会人发布端 | 新增"报名赠送 1.99 元 TaoToken 额度"勾选选项（默认勾选） |
| 参会人报名端 | 报名按钮下方新增 TaoToken 协议入口；报名时后台静默注册并发放额度 |
| 签到系统 | 参会人签到后触发 3.99 元额外额度发放 |
| 会议详情页 | 底部新增 TaoToken 广告区域 |
| 后端 | 新增 TaoToken API 调用模块（注册、发放额度）+ 异步重试队列 |

### 1.5 前置依赖

### 2.2 办会人端赠送开关（TT-01）

#### 场景描述

办会人在发布会议时，在确认创建步骤之前，系统展示一个"报名赠送 TaoToken 额度"的勾选项，默认勾选。办会人可根据需要取消勾选。此开关决定该会议下的参会人是否能在报名和签到时获得 TaoToken 额度奖励。

#### 用户故事

| 编号 | 用户故事 |
|------|----------|
| US-TT-01 | 作为办会人，我希望在发布会议时看到"报名赠送 1.99 元 TaoToken 额度"选项，默认勾选，以便为技术参会人提供额外福利 |
| US-TT-02 | 作为办会人，我希望可以取消勾选此选项，以便对不需要此福利的会议不做赠送 |

#### 需求规格

| 规则项 | 内容 |
|--------|------|
| 配置位置 | 快速创建会议 / 完整创建会议流程中「确认创建」步骤之前，作为独立选项 |
| 选项文案 | 勾选框 + "报名赠送 1.99 元 TaoToken 额度" |
| 默认状态 | **勾选**（默认赠送） |
| 可编辑性 | 会议发布前可修改；会议发布后**可以修改**（后端编辑接口） |
| 编辑影响 | 办会人取消勾选后：已通过报名领取的额度**不追回**；新报名的参会人不再发放；签到也不再发放 3.99 元 |
| 会议详情页展示 | 若办会人勾选了此选项，会议详情页报名按钮区域展示 TaoToken 赠送提示 |

#### 验收标准

- [ ] AC-TT-01：办会人创建会议时，默认勾选"报名赠送 1.99 元 TaoToken 额度"
- [ ] AC-TT-02：办会人可取消勾选
- [ ] AC-TT-03：取消勾选后，新报名的参会人不再获得额度（报名和签到均不赠送）
- [ ] AC-TT-04：取消勾选后，已领取的额���不追回

### 2.3 参会人报名 + 自动注册发放（TT-02）

#### 场景描述

参会人在会议详情页点击报名按钮时，报名区域底部在现有的"隐私政策"和"用户协议"之外，新增 TaoToken 协议入口。参会人提交报名后，会议系统以参会人手机号为关联键，在后台调用 TaoToken 注册接口静默创建/关联账号，并调用额度发放接口自动发放 1.99 元额度。

#### 用户故事

| 编号 | 用户故事 |
|------|----------|
| US-TT-03 | 作为参会人，我报名会议时希望看到 TaoToken 协议信息，以便了解赠送的额度相关条款 |
| US-TT-04 | 作为参会人，我报名成功后希望自动获得 TaoToken 额度，无需额外操作 |

#### 需求规格

**2.3.1 报名按钮下方协议区域**

| 规则项 | 内容 |
|--------|------|
| 展示位置 | 报名按钮下方，现有"隐私政策"和"用户协议"之后 |
| 展示文案 | "报名即表示您同意《TaoToken 用户协议》"（含可点击链接） |
| 链接行为 | 点击后弹窗或以新页签展示 TaoToken 协议全文 |
| 协议文档来源 | 待甲方提供；嵌入方式（内嵌链接 vs 站内全文）待定 |

**2.3.2 静默注册与额度发放流程**

```
参会人提交报名
  ↓
系统校验报名数据通过
  ↓
写入报名记录，状态 = 待审核
  ↓
检查：当前会议是否开启"报名赠送 TaoToken 额度"？
  ↓ 是
将发放任务写入异步队列（phone, meeting_id, amount=1.99, trigger=registration）
  → 返回报名成功（不等待 TaoToken 接口返回）
  ↓
异步 Worker 消费队列：
  1. 调用 TaoToken 注册/查询接口（以手机号）
  2. 调用 TaoToken 额度发放接口（1.99 元）
  3. 成功后记录发放日志
  4. 失败后标记 retry_count，隔 5 分钟重试，最多 3 次
  5. 3 次均失败 → 记录为最终失败，触发人工/自动告警
```

| 规则项 | 内容 |
|--------|------|
| 账号关联键 | 手机号 |
| 发放金额 | 1.99 元 |
| 发放时机 | 报名提交成功即触发（与审核状态无关） |
| 发放策略 | **异步：不阻断报名流程。** TaoToken 接口调用在后台异步执行 |
| 重复发放防护 | 同一手机号 + 同一会议 ID + 同一触发类型（registration）仅发放一次 |
| 开关检查 | 仅在办会人勾选了赠送选项时触发；未勾选不触发 |
| 失败重试 | 间隔 5 分钟，最多重试 3 次。3 次均失败后标记为最终失败，记录日志并告警 |

**2.3.3 TaoToken API 接口约定（待双方开发对齐）**

| 接口 | 方法 | 参数 | 返回 |
|------|------|------|------|
| 注册/查询用户 | POST | `{ phone: "138xxxx" }` | `{ user_id: "xxx", is_new: true/false }` |
| 发放额度 | POST | `{ user_id: "xxx", amount: 1.99, source: "meeting_registration", meeting_id: "xxx" }` | `{ success: true, transaction_id: "xxx" }` |

> **注**：以上为建议接口，实际以双方开发对齐为准。

#### 验收标准

- [ ] AC-TT-05：报名按钮下方展示 TaoToken 协议入口（文案 + 点击查看链接）
- [ ] AC-TT-06：报名提交成功后，后台自动调用 TaoToken 注册 + 额度发放接口
- [ ] AC-TT-07：TaoToken 接口调用失败不阻塞报名流程，参会人仍正常完成报名
- [ ] AC-TT-08：同一手机号 + 同一会议 + 报名类型不重复发放
- [ ] AC-TT-09：会���未勾选赠送时，报名不触发额度发放

### 2.4 签到奖励发放（TT-03）

#### 场景描述

参会人完成签到后（状态变为"已签到"），系统自动调用 TaoToken 接口额外发放 3.99 元额度。仅当该会议的办会人勾选了"报名赠送 TaoToken 额度"时触发。

#### 用户故事

| 编号 | 用户故事 |
|------|----------|
| US-TT-05 | 作为参会人，我签到成功后希望额外获得 TaoToken 额度，以便鼓励真实到场 |

#### 需求规格

| 规则项 | 内容 |
|--------|------|
| 发放金额 | 3.99 元 |
| 触发条件 | 参会人签到状态变为"已签到"（含扫码签到和手动签到） |
| 前置条件 | 该会议的办会人勾选了"报名赠送 1.99 元 TaoToken 额度"选项。**未勾选则签到也不赠送** |
| 发放策略 | 同 2.3.2：异步队列、不阻断签到流程、失败重试 3 次 |
| 重复发放防护 | 同一手机号 + 同一会议 ID + 触发类型（checkin）仅发放一次 |
| 独立于报名奖励 | 报名已发放的 1.99 元与签到赠送的 3.99 元是两次独立发放，分别校验去重 |
| 签退/取消签到 | 当前不支持取消签到，无此场景 |

#### 验收标准

- [ ] AC-TT-10：参会人签到完成后，后台自动调用 TaoToken 额度发放接口（3.99 元）
- [ ] AC-TT-11：扫码签到和手动签到均触发
- [ ] AC-TT-12：办会人未勾选赠送时，签到不触发额度发放
- [ ] AC-TT-13：同一手机号 + 同一会议 + 签到类型不重复发放

### 2.5 TaoToken API 集成（TT-04）

#### 场景描述

会议系统后端与 TaoToken 系统对接，实现用户注册/查询和额度发放两个核心接口的调用、异步重试和日志记录。

#### 需求规格

| 规则项 | 内容 |
|--------|------|
| 接口鉴权 | 待双方开发对齐（API Key / OAuth Token），凭据通过环境变量管理，不写入代码或日志 |
| 连接超时 | 5 秒 |
| 异步队列 | 使用现有后端消息队列组件（如 Redis Queue / BullMQ），独立 Worker 消费 |
| 重试策略 | 失败间隔 5 分钟，最多 3 次；3 次均失败标记永久失败 |
| 失败日志 | 记录：手机号、会议 ID、发放金额、触发类型、失败原因、重试次数、时间戳 |
| 告警 | 连续失败或日失败率超过阈值时触发告警（阈值待运维配置） |
| 幂等性 | 发放前检查 `(phone, meeting_id, trigger_type)` 是否已有成功记录，防重复 |

#### 验收标准

- [ ] AC-TT-14：TaoToken 接口调用成功时，发放记录写入日志
- [ ] AC-TT-15：接口调用失败时自动重试，3 次均失败标记永久失败并记录日志
- [ ] AC-TT-16：同一发放任务不会重复执行（幂等）

### 2.6 详情页广告（TT-05）

#### 场景描述

每个会议详情页底部新增一个 TaoToken 广告区域，吸引参会人了解和前往 TaoToken 系统。

#### 需求规格

| 规则项 | 内容 |
|--------|------|
| 展示位置 | 会议详情页底部（会议信息与地图之后、报名悬浮按钮之前） |
| 展示范围 | **所有会议**详情页均展示（不区分会议类型或是否勾选赠送选项） |
| 广告形式 | 横幅 Banner，固定内容 |
| 广告内容（示例） | 标题："TaoToken — 大模型额度中转站"；描述："一站调用 GPT-4、文心一言等大模型，注册即送体验额度"；按钮："立即体验" |
| 点击行为 | 跳转 TaoToken 系统首页（URL 待甲方提供） |
| 配置方式 | **首版硬编码**，不提供运营端配置能力。后续可迭代为可配置广告模块 |


#### 验收标准

- [ ] AC-TT-17：所有会议详情页底部展示 TaoToken 广告 Banner
- [ ] AC-TT-18：点击 Banner 或按钮跳转 TaoToken 系统首页

## 第6章：风险与依赖

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| TaoToken API 不可用 | 额度发放全部失败 | 异步队列 + 重试 + 告警机制；不影响报名和签到主流程 |
| 手机号作为关联键导致匹配失败（如 TaoToken 侧手机号格式不一致） | 部分用户注册/发放失败 | 双方对齐手机号格式（统一 `+86` 或纯数字）；日志中记录原始值便于排查 |
| 办会人默认勾选赠送但未告知参会人 | 参会人收到 TaoToken 额度感到困惑 | 报名区域展示赠送提示 + TaoToken 协议说明 |
