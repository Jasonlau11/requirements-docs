# 注册流程简化产品需求说明书

## 需求概览

将现有表单注册流程从"头像+昵称+手机号+邮箱+密码+确认密码"全量输入，简化为仅保留"手机号（必填）+短信验证码（必填）+邮箱（选填）+邮箱验证码（填邮箱时触发）"。头像和昵称由系统在注册时自动生成默认值，密码字段不出现、初始密码为空，用户可在登录后自行设置。注册成功后直接进入已登录态，无需二次登录。

本次表单字段简化范围仍为**表单注册路径（2.2.1）**；同时补充一条通用注册风控规则：凡会创建本站新账号的注册路径，均需在提交注册时执行设备注册校验。CSDN App 扫码登录路径不做改动；若扫码授权触发新建本站账号，则同样纳入设备注册校验。

### 1.3 背景和价值

**背景问题**：当前表单注册要求用户同时完成头像选择、昵称输入、密码设置与确认等多步操作，注册表单信息密度高、填写阻力大。尤其在"未登录→触发需登录操作→弹窗注册"的场景下，冗余字段增加了用户的决策成本和填写时长，直接推高注册流失率。甲方反馈当前注册弹框样式不够专业，要求简化。

**业务价值**：
- **降低注册阻力**：从 6-7 个必填字段削减至 2 个必填字段（手机号+验证码），大幅缩短注册完成时间。
- **提升转化率**：减少报名、收藏等场景下的注册弹框流失，将更多"想报名"的用户转化为注册用户。
- **后续可运营**：用户登录后可自主完善昵称、头像、设置密码，平台可通过引导和触达推动用户完善资料，而非在注册关口设阻。

**用户价值**：
- 快速完成注册，不因繁琐填表而放弃操作意图。
- 系统自动生成的身份可满足基本展示需求，不影响注册后立即使用平台功能。
- 昵称、头像均可后续自由修改，密码可随时设置，保持自主控制权。

## 第2章：功能需求

### 2.1 注册表单简化（REG-SIMP-01）

#### 场景描述

用户在未登录状态下触发需登录操作（如报名、收藏），系统弹出注册/登录弹窗。用户切换到"注册"标签页，所见注册表单仅含：手机号（必填）、短信验证码（必填）、邮箱（选填）、邮箱验证码（填写邮箱时触发）。头像、昵称、密码均不出现在注册表单中。

用户填写手机号并通过短信验证后，可选填邮箱（若填写则需完成邮箱验证）；勾选《用户协议》和《隐私政策》两项同意后点击"注册"，系统创建账号、自动分配默认头像和默认昵称（密码初始为空），并直接建立已登录态。注册成功后展示"立即完善身份信息"或"暂不完善"的可选步骤。

#### 用户故事

| 编号 | 用户故事 |
|------|----------|
| US-REG-SIMP-01 | 作为新用户，我希望注册时只需填写手机号和验证码即可快速完成注册，以便减少填表阻力、不因繁琐表单而放弃操作意图 |
| US-REG-SIMP-02 | 作为新用户，我希望注册时可以选择填写邮箱（非强制），以便根据自身意愿决定是否绑定邮箱 |
| US-REG-SIMP-03 | 作为新用户，我希望注册成功后系统自动为我分配一个可辨识的默认昵称和头像，以便不影响我立即浏览和使用平台 |
| US-REG-SIMP-04 | 作为新用户，我希望注册成功后自动变为已登录状态而不需要再登录一次，以便无缝继续刚才的操作意图 |
| US-REG-SIMP-05 | 作为已注册用户，我可以在个人中心自行设置密码，以便使用密码方式登录并保障账户安全 |
| US-REG-SIMP-06 | 作为已注册用户，我可以在个人中心修改昵称和头像，以便将默认身份替换为我想要的个性化展示 |

#### 需求规格

**2.1.1 注册表单字段变更**

简化后注册表单仅保留以下字段：

| 字段 | 类型 | 必填 | 交互规则 |
|------|------|------|----------|
| 手机号 | 文本输入 | 是 | 中国大陆手机号格式校验；输入完成后方可点击"获取验证码" |
| 短信验证码 | 数字输入 | 是 | 点击"获取验证码"后发送；有效期与频控沿用现有规则；验证通过后视为手机号归属确认 |
| 邮箱 | 文本输入 | 否 | 选填字段；若用户填写邮箱，则需点击"获取邮箱验证码"并输入正确验证码；若用户不填邮箱，跳过此字段即可提交 |
| 邮箱验证码 | 数字输入 | 条件必填 | 仅当用户填写了邮箱时显示和必填；发送规则与频控沿用现有规则 |
| 用户协议 + 隐私政策 | 勾选 | 是 | 注册必要条件，须两项均勾选同意后方可提交；任一未勾选则阻止提交且不创建账号 |

**不再出现于注册表单的字段**（无需用户操作，由系统自动处理）：

| 字段 | 处理方式 |
|------|----------|
| 头像 | 注册成功时从预设默认头像库中随机选取一个，写入 `avatar_url` |
| 昵称 | 注册成功时自动生成，格式为"CSDN参会者_XXXX"（XXXX 为4位十六进制随机码），写入 `nickname` |
| 登录密码 | 注册时密码字段不出现，初始密码为空；用户可在登录后通过个人中心设置密码 |
| 确认密码 | 无需，因为注册表单不展示密码字段 |

**2.1.2 默认昵称生成规则**

- **命名格式**：`CSDN参会者_` + 4位十六进制随机码
- **示例**：`CSDN参会者_7a3f`、`CSDN参会者_b02e`、`CSDN参会者_d19c`
- **唯一性保障**：系统在生成随机码后校验当前用户表中是否已存在相同昵称；若冲突则递增随机码或重新随机直至唯一，最多重试 100 次（极端情况下超出重试次数则使用 `CSDN参会者_` + 精确到秒的时间戳后4位十六进制作为退路方案）
- **可修改性**：用户登录后可在"个人中心"自由修改昵称，修改规则沿用现有昵称校验规则（长度、字符约束）
- **展示策略**：默认昵称在平台各展示场景下与用户自定义昵称无差异化对待，不附加特殊标记

**2.1.3 默认头像选取规则**

- **来源**：从现有预设默认头像库中随机选取一张
- **选取时机**：注册成功、账号创建时一次性确定
- **种子策略**：以用户 ID 或注册时间戳作为随机种子，保证有据可查但不可预测
- **可更换性**：用户登录后可在"个人中心"上传自定义头像或从默认库重新选择，规则沿用现有头像管理规则
- **扫码注册兼容**：CSDN App 扫码触发新账号注册时，头像仍由 CSDN 返回或用户手动选择；本次仅增加设备注册校验，不改变头像补充规则

**2.1.4 初始密码为空**

- **初始状态**：用户注册时密码字段不出现在表单中，账号创建后密码为空（或存为 NULL/空值）
- **登录方式**：用户注册成功后已处于已登录态，正常使用无需密码。后续登录可通过"手机号+短信验证码"完成；登录后在个人中心设置密码后，即可使用"手机号+密码"登录
- **密码设置**：用户在个人中心 → 账号安全中设置密码，需验证当前手机号短信验证码后设置新密码，密码须满足现有强度规则（长度≥8位，含大写字母、小写字母、数字）
- **首次设置 vs 后续修改**：首次设置密码（密码从空变为有值）与后续修改密码（密码已有值）使用同一入口和流程，不做区分

**2.1.5 协议与隐私同意**

- 规则与现有用户体系 PRD 保持一致：须同时同意《用户协议》与《隐私政策》后方可提交注册
- 若任一未勾选，提交按钮被拦截，系统不创建账号、不建立登录态
- 两项协议的链接文本可点击，在新标签页打开完整协议内容

**2.1.6 手机号已注册提示**

- 用户在注册表单中输入手机号并完成短信验证码校验后，若系统检测到该手机号已在本站注册：
  - 提交注册时拦截，提示"该手机号已注册，请直接登录"，并自动切换到登录标签页（或提供快捷跳转）
  - 不创建重复账号

**2.1.7 设备注册校验**

为防止用户通过多个手机号重复注册账号并反复获取办会人首次免费发布额度，注册提交时需增加设备校验。该校验仅作用于新账号注册，不限制同一设备登录多个已有账号。

| 规则项 | 内容 |
|------|------|
| 校验时机 | 用户完成手机号验证码、邮箱验证码（如填写邮箱）、协议与隐私政策勾选后，点击"注册"提交时触发 |
| 识别维度 | 综合 IP 地址、浏览器 User-Agent、设备指纹、操作系统信息、浏览器信息、Cookie / LocalStorage 标识等生成设备识别结果；具体采集范围以技术方案与合规要求为准 |
| 注册上限 | 同一设备最多注册 2 个账号 |
| 超限处理 | 若当前设备已注册账号数量 ≥ 2，则阻止提交，不创建账号，不建立登录态 |
| 提示文案 | 当前设备注册账号数量已达上限，请使用已有账号登录或联系管理员 |
| 登录边界 | 不限制同一设备登录多个已有账号；用户在登录页使用手机号验证码、手机号密码或 CSDN App 扫码登录时，不因设备注册数量触发拦截 |
| 记录要求 | 注册成功或注册被设备规则拦截时，系统需记录设备识别结果、注册手机号、IP、User-Agent、触发时间、处理结果，便于风控复盘和误拦截排查 |
| 合规要求 | 设备信息采集应在隐私政策覆盖范围内，仅用于注册风控与反滥用；前端无需向用户暴露设备指纹细节 |

#### 业务流程

```
用户进入注册弹窗
  ↓
输入手机号 → 获取短信验证码 → 输入验证码 → 系统校验通过
  ↓
（可选）填写邮箱 → 获取邮箱验证码 → 输入验证码 → 系统校验通过
  ↓
勾选《用户协议》+《隐私政策》
  ↓
点击"注册"
  ↓（手机号已注册？）
  │─ 是 → 提示"该手机号已注册，请直接登录"，切换至登录页
  │─ 否 → 进入设备注册校验
           ↓（当前设备已注册账号数 ≥ 2？）
           │─ 是 → 提示"当前设备注册账号数量已达上限，请使用已有账号登录或联系管理员"，不创建账号
           │─ 否 → 系统创建账号：
  │        - 写入手机号
  │        - 写入邮箱（若填写且验证通过）
  │        - 自动生成默认昵称
  │        - 随机选取默认头像
  │        - 密码初始为空
  │        - 建立已登录态
  ↓
注册成功提示
  ↓
展示可选步骤：立即完善身份信息 / 暂不完善，继续浏览
  ↓
关闭弹窗，继续原操作意图
```

## 第3章：本期不包含（Non-Goals）

1. **不修改 CSDN App 扫码登录**：扫码登录路径保持不变；若扫码授权触发新建本站账号，则同样执行设备注册校验，防止绕过表单注册限制。
2. **不新增密码找回/重置功能**：若一期尚未实现密码找回，"手机号+短信验证码登录"作为兜底方案可满足登录需求；密码找回为独立需求，不在本次简化范围。
3. **不修改登录流程**：登录方式（手机号+密码、手机号+验证码、CSDN App 扫码）保持不变。
4. **不新增注册成功后的引导弹窗**：注册成功后除保留现有的"完善身份信息"可选步骤外，不新增额外的引导步骤（如"立即修改头像"、"立即设置密码"等）。
5. **不开放邮箱注册作为主注册方式**：手机号仍为注册的唯一主标识，邮箱仅作为可选的辅助联系方式绑定。
6. **不修改个人资料维护页面**：个人中心中昵称、头像、密码的修改功能保持不变，无需额外开发。
