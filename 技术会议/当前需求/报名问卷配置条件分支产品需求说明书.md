# 报名问卷配置条件分支产品需求说明书

## 需求概览

在现有报名问卷配置能力基础上，新增**条件分支（下钻）**能力：办会人配置单选/下拉问题时，可为每个选项挂载一组跟随问题；参会人选择某一选项后，系统动态展示该选项对应的跟随问题，未选中选项的跟随问题完全不渲染。最多支持 2 层嵌套（根问题→跟随问题），跟随问题中的单选/下拉不再支持继续下钻。

---

## 第1章：概述

### 1.1 术语表

| 名称 | 详细描述 |
|------|----------|
| 条件分支 / 下钻 | 根据参会人对单选/下拉问题的选项选择，动态展示/隐藏对应跟随问题的交互逻辑 |
| 根问题 | 挂载了跟随问题的单选/下拉字段（问题本身） |
| 跟随问题 | 挂载在根问题某个选项下的子问题，仅在参会人选中该选项时展示 |
| 嵌套层级 | 根问题为第 1 层，跟随问题为第 2 层；跟随问题中的单选/下拉不支持继续挂载跟随问题 |

|------|------|------|------|------|

### 1.3 背景和价值

- **背景问题**：现有报名问卷是平铺的字段列表（字段名、类型、必填规则），所有字段始终可见，无法根据参会人的前置选择动态调整问卷内容。当办会人需要区分人群收集不同信息时（如"是否住宿"→是→展开住宿偏好问题），只能把所有字段摊开让参会人自行跳过，体验差且数据质量低。
- **业务价值**：提升报名问卷的灵活性与精准度，让办会人能设计更专业的报名流程；减少无关字段对参会人的干扰，降低填写放弃率。
- **用户价值**：办会人可根据选项条件精准收集信息；参会人只看到与自己相关的字段，填写路径清晰。

## 第2章：功能需求

### 2.1 条件分支配置（COND-01）

#### 场景描述

办会人在配置报名问卷时，对"单选"或"下拉"类型的字段，可为每个选项配置 0~N 个跟随问题。每个跟随问题是一个完整的独立字段（支持全部字段类型）。参会人实际填写时，仅当选中某个选项后，其对应的跟随问题才动态出现在页面上。

#### 用户故事

| 编号 | 用户故事 |
|------|----------|
| US-COND-01 | 作为办会人，我希望给单选/下拉问题的每个选项挂载不同的跟随问题，以便根据参会人的选择收集差异化信息 |
| US-COND-02 | 作为办会人，我希望跟随问题支持全部字段类型（文本、长文本、单选框组、多选框组、下拉选择），以便灵活设计问卷 |
| US-COND-03 | 作为办会人，我希望跟随问题同样支持配置"是否必填"，以便根据业务需要强制收集关键信息 |
| US-COND-04 | 作为参会人，我选择某个选项后，希望页面自动展示该选项对应的跟随问题，不相关的字段不要显示，以便填写体验清晰 |
| US-COND-05 | 作为参会人，我切换选项后，希望上一个选项的跟随问题自动隐藏，新选中选项的跟随问题自动展示，以便交互符合直觉 |

#### 需求规格

**2.1.1 配置端交互**

```
办会人进入报名问卷配置页面
  ↓
选中某个"单选"或"下拉"字段
  ↓
在该字段的每个选项右侧展示【+添加跟随问题】入口
  ↓
点击【+添加跟随问题】→ 在当前选项下方展开跟随问题编辑区
  ↓
跟随问题编辑区内：
  - 支持添加多个跟随问题（字段类型不限）
  - 每个跟随问题可配置：名称、类型、是否必填
  - 跟随问题之间可拖拽排序（P1）
  ↓
保存问卷配置，生成含条件分支结构的 form_schema
```

| 规则项 | 内容 |
|--------|------|
| 适用范围 | 仅"单选"和"下拉"类型字段可挂载跟随问题；"多选""文本""长文本"不支持 |
| 最大嵌套 | 2 层（根问题 + 跟随问题）；跟随问题中的单选/下拉**不支持**继续挂载跟随问题，对应的【+添加跟随问题】入口不展示 |
| 跟随问题数量 | 每个选项下 0~N 个，无上限 |
| 跟随问题类型 | 支持全部类型：文本、长文本、单选框组、多选框组、下拉选择 |
| 是否必填 | 跟随问题独立配置必填/非必填，与根问题无关 |

**2.1.2 参会端渲染逻辑**

| 情形 | 根问题状态 | 页面渲染 |
|------|-----------|----------|
| 未选择任何选项 | 无选中值 | 仅展示根问题；所有选项的跟随问题均不渲染 |
| 选中选项 A | 值为 A | 展示根问题 + 选项 A 的跟随问题；选项 B/C/D...的跟随问题不渲染 |
| 从 A 切换到 B | 值由 A → B | A 的跟随问题隐藏，B 的跟随问题展示；参会人已在 A 的跟随问题中填写的内容**清空** |
| 清空选择 | 值置空 | 所有跟随问题隐藏，已填写内容清空 |

#### 验收标准

- [ ] AC-COND-01：办会人在问卷配置页，对单选/下拉字段的每个选项，能看到【+添加跟随问题】入口
- [ ] AC-COND-02：办会人点击入口后，能在选项下方展开编辑区，添加多个跟随字段（文本/长文本/单选/多选/下拉）
- [ ] AC-COND-03：跟随问题中的单选/下拉字段，不展示【+添加跟随问题】入口（阻止第 3 层嵌套）
- [ ] AC-COND-04：多选/文本/长文本字段，不展示【+添加跟随问题】入口
- [ ] AC-COND-05：参会人未选择根问题选项时，页面上不展示任何跟随问题
- [ ] AC-COND-06：参会人选择选项 A 后，选项 A 的跟随问题动态出现在根问题下方
- [ ] AC-COND-07：参会人从选项 A 切换到选项 B 后，A 的跟随问题消失、B 的跟随问题出现、A 中已填内容被清空
- [ ] AC-COND-08：参会人清空根问题选项后，所有跟随问题消失
- [ ] AC-COND-09：跟随问题标记为必填时，参会人未填写则提交时校验失败并给出提示

### 2.2 条件分支数据结构（COND-DATA）

#### 需求规格

现有 `form_schema` 为平铺数组。条件分支引入后，字段增加可选的 `options_branches` 属性，用于定义选项与跟随问题的映射。

```json
// 示例：根问题的数据结构
{
  "field_id": "f_001",
  "field_name": "是否需要住宿",
  "field_type": "radio",
  "required": true,
  "options": [
    {"value": "yes", "label": "是"},
    {"value": "no", "label": "否"}
  ],
  "options_branches": {
    "yes": [
      {
        "field_id": "f_002",
        "field_name": "入住日期",
        "field_type": "text",
        "required": true,
        "options": [],
        "options_branches": {}
      },
      {
        "field_id": "f_003",
        "field_name": "房型偏好",
        "field_type": "radio",
        "required": false,
        "options": [
          {"value": "single", "label": "单人间"},
          {"value": "double", "label": "双人间"}
        ],
        "options_branches": {}
      }
    ],
    "no": []
  }
}
```

| 规则项 | 内容 |
|--------|------|
| `options_branches` | 可选属性；key 为选项 value，value 为该选项对应的跟随问题数组 |
| 跟随问题数组 | 与平铺字段结构一致（field_id, field_name, field_type, required, options）；跟随问题的 options_branches 恒为空对象（不支持继续分支） |
| 前端渲染 | 按 options_branches 的 key 匹配当前选中值，渲染对应数组中的字段 |

## 第3章：Non-Goals（本期不包含）

| 序号 | 不包含内容 | 说明 |
|------|-----------|------|
| 1 | 第 3 层嵌套 | 跟随问题中的单选/下拉不支持继续挂载跟随问题，仅支持根→跟随共 2 层 |
| 2 | 多选字段挂载跟随问题 | 仅单选/下拉支持条件分支，多选因选中状态组合复杂，暂不支持 |
| 3 | 跟随问题的拖拽排序 | P1 优先级，首版可后续迭代 |
| 4 | 条件分支逻辑的导出/导入 | 首版不涉及问卷模板的跨会议复用 |
| 5 | 跟随问题的条件逻辑（跨字段联动） | 仅支持单选/下拉的选项驱动，不支持"字段 A 的值为 X 时展示字段 B"的通用条件引擎 |

## 第4章：成功指标

| 指标 | 类型 | 目标值 | 衡量方式 |
|------|------|--------|----------|
| 条件分支功能使用率 | 领先 | 上线 30 天内，≥15% 的新建会议配置了至少 1 个带分支的单选/下拉字段 | 统计 form_schema 中含 options_branches 的会议占比 |
| 跟随问题填写完成率 | 领先 | ≥90%（曝光即填写完成） | 跟随问题渲染次数 vs 有效提交次数 |
| 问卷放弃率 | 滞后 | 不高于平铺问卷的放弃率 | 对比同期平铺问卷与含分支问卷的提交率 |
